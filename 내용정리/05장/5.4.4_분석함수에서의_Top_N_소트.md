## 5.4.4 분석함수에서의 Top N 소트

### 핵심 개념

분석함수에서도 Top N 소트가 작동할 수 있습니다. 특히 `rank`, `row_number`처럼 순위를 매기는 함수는 `max` 분석함수보다 소트 부하가 적을 수 있습니다.

```text
max() over (...)
   ↓
파티션 전체 윈도우 계산
   ↓
WINDOW SORT 부담 큼

rank()/row_number() over (order by ...)
   ↓
상위 순위만 필요
   ↓
WINDOW SORT PUSHED RANK
   ↓
Top N 소트 효과
```

### max 분석함수 방식

아래 쿼리는 장비별 마지막 변경순번을 `max` 분석함수로 구합니다.

```sql
select 장비번호, 변경일자, 변경순번, 상태코드, 메모
from (
  select 장비번호, 변경일자, 변경순번, 상태코드, 메모,
         max(변경순번) over (partition by 장비번호) 최종변경순번
  from 상태변경이력
  where 변경일자 = :upd_dt
)
where 변경순번 = 최종변경순번;
```

실행계획에는 일반 `WINDOW SORT`가 나타납니다.

```text
VIEW
  WINDOW SORT
    TABLE ACCESS FULL 상태변경이력
```

Sort Area가 작으면 Window Sort 단계에서 많은 Temp Read/Write가 발생할 수 있습니다.

### rank 방식

같은 문제를 `rank`로 풀면 Top N 성격이 생깁니다.

```sql
select 장비번호, 변경일자, 변경순번, 상태코드, 메모
from (
  select 장비번호, 변경일자, 변경순번, 상태코드, 메모,
         rank() over (
           partition by 장비번호
           order by 변경순번 desc
         ) rnum
  from 상태변경이력
  where 변경일자 = :upd_dt
)
where rnum = 1;
```

실행계획은 다음처럼 나타날 수 있습니다.

```text
VIEW
  WINDOW SORT PUSHED RANK
    TABLE ACCESS FULL 상태변경이력
```

`WINDOW SORT PUSHED RANK`는 파티션별 상위 순위만 유지하는 방식으로 작동하므로 일반 `WINDOW SORT`보다 소트 부하가 작습니다.

### 비교

| 방식 | 실행계획 | 특징 |
|---|---|---|
| `max() over` | `WINDOW SORT` | 파티션 전체 값을 계산하고 비교 |
| `rank() over` | `WINDOW SORT PUSHED RANK` | 순위 조건이 밀려 들어가 Top N 소트 효과 |
| `row_number() over` | `WINDOW SORT PUSHED RANK` 가능 | 1건만 필요할 때 특히 유용 |

### 튜닝 포인트

분석함수는 편리하지만, 함수 선택에 따라 소트 부하가 달라집니다.

```text
최신 1건 필요
   ↓
max 분석함수로 전체 파티션 계산?
   ↓
rank/row_number로 상위 1건만 유지?
   ↓
WINDOW SORT PUSHED RANK 가능성 확인
```

---

## ✅ 이 절(5.4.4)의 핵심 통찰

1. 분석함수도 큰 소트를 유발할 수 있습니다.
2. `max() over`는 일반 `WINDOW SORT`로 큰 Temp I/O를 만들 수 있습니다.
3. `rank`, `row_number`는 `WINDOW SORT PUSHED RANK`로 Top N 소트 효과를 얻을 수 있습니다.
4. 최신 1건 조회라면 순위 함수 방식이 더 유리할 수 있습니다.
5. 실행계획에서 `WINDOW SORT`와 `WINDOW SORT PUSHED RANK`를 구분해야 합니다.

## 🔗 흐름 속 위치

5.4.2와 5.4.3에서 본 Top N 소트 원리가 분석함수에도 적용됩니다. 5.4.4는 5장의 마지막 절로, 소트 부하를 줄이는 관점이 일반 정렬뿐 아니라 분석함수까지 확장된다는 것을 보여줍니다.

📌 한 줄 요약: 분석함수에서는 `max`보다 `rank`/`row_number`가 Top N 소트를 유도해 소트 부하를 줄일 수 있습니다.
