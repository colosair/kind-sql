## 5.3.5 Sort Group By 생략

### 핵심 개념

`ORDER BY`뿐 아니라 `GROUP BY`도 인덱스를 이용해 소트 연산을 생략할 수 있습니다. 그룹핑 컬럼이 인덱스 선두에 있으면 같은 그룹의 데이터가 인덱스에서 연속으로 모여 있기 때문입니다.

```text
인덱스: region
GROUP BY region
   ↓
region 값별로 연속 스캔
   ↓
그룹 경계가 바뀌는 순간 이전 그룹 집계 완료
   ↓
SORT GROUP BY NOSORT
```

실행계획에는 다음처럼 표시됩니다.

```text
SORT GROUP BY NOSORT
  TABLE ACCESS BY INDEX ROWID
    INDEX FULL SCAN
```

### 처리 과정

예를 들어 `region`별 평균 나이와 건수를 구한다고 하겠습니다.

```sql
select region, avg(age), count(*)
from customer
group by region;
```

`region`이 선두 컬럼인 인덱스를 사용하면 다음처럼 처리할 수 있습니다.

```text
region = A 구간 스캔
   ↓
A 집계 완료
   ↓
region = B를 만나는 순간 A 결과 반환 준비
   ↓
B 구간 집계
   ↓
region = C를 만나는 순간 B 결과 반환 준비
```

Array Size가 3이면 A, B, C 세 그룹 결과를 클라이언트로 전송한 뒤 다음 Fetch를 기다릴 수 있습니다.

### 부분범위 처리

일반적인 `GROUP BY`는 전체 입력을 처리한 뒤 결과를 반환한다고 생각하기 쉽습니다. 하지만 `GROUP BY NOSORT`는 인덱스 순서 덕분에 그룹 경계가 확정되는 순간 결과를 반환할 수 있습니다.

```text
Sort Group By
   ↓
전체 또는 많은 그룹 상태 처리
   ↓
결과 반환

Group By Nosort
   ↓
인덱스에서 같은 그룹 연속 스캔
   ↓
그룹 경계마다 결과 반환
   ↓
부분범위 처리 가능
```

---

## ✅ 이 절(5.3.5)의 핵심 통찰

1. 그룹핑 컬럼이 인덱스 선두에 있으면 `GROUP BY` 소트를 생략할 수 있습니다.
2. 실행계획에는 `SORT GROUP BY NOSORT`가 나타납니다.
3. 같은 그룹의 데이터가 인덱스에서 연속되므로 그룹 경계마다 집계를 완료할 수 있습니다.
4. `GROUP BY NOSORT`는 부분범위 처리도 가능하게 합니다.
5. 인덱스 정렬성은 `ORDER BY`뿐 아니라 `GROUP BY`에도 활용됩니다.

## 🔗 흐름 속 위치

5.3.1이 `ORDER BY` 소트 생략이었다면, 5.3.5는 같은 인덱스 정렬성을 그룹핑에 적용합니다.

📌 한 줄 요약: 그룹핑도 정렬된 인덱스를 이용하면 전체 소트 없이 그룹 경계만 감지하며 처리할 수 있습니다.
