## 5.3.2 Top N 쿼리

### 핵심 개념

Top N 쿼리는 전체 결과집합 중 상위 N건만 선택하는 쿼리입니다. 인덱스 순서가 `ORDER BY`와 맞으면 오라클은 N건을 읽는 순간 멈출 수 있습니다. 이를 `Top N Stopkey` 알고리즘이라고 합니다.

```text
정렬 순서에 맞는 인덱스
   ↓
인덱스 Range Scan
   ↓
ROWNUM <= N 조건
   ↓
N건 읽는 순간 중단
```

오라클 전통 방식의 Top N 쿼리는 다음과 같습니다.

```sql
select *
from (
  select 거래일시, 체결건수, 체결수량, 거래대금
  from 종목거래
  where 종목코드 = 'KR123456'
  and 거래일시 >= '20180304'
  order by 거래일시
)
where rownum <= 10;
```

실행계획에서는 `COUNT (STOPKEY)`가 핵심 신호입니다.

```text
SELECT STATEMENT
  COUNT STOPKEY
    VIEW
      TABLE ACCESS BY INDEX ROWID
        INDEX RANGE SCAN
```

### 3-Tier 환경과 페이징

3-Tier 환경에서는 DB 커넥션을 오래 붙잡고 클라이언트가 추가 Fetch를 요청할 때마다 조금씩 반환하는 전통적 부분범위 처리가 어렵습니다. 하지만 Top N 페이징을 사용하면 부분범위 처리 원리를 여전히 활용할 수 있습니다.

표준 페이징 패턴은 다음과 같습니다.

```sql
select *
from (
  select rownum no, a.*
  from (
    /* SQL Body */
  ) a
  where rownum <= (:page * 10)
)
where no >= (:page - 1) * 10 + 1;
```

핵심은 안쪽의 `rownum <= (:page * 10)`입니다. 이 조건이 있어야 `STOPKEY`가 작동합니다.

```text
1페이지 요청
   ↓
rownum <= 10
   ↓
10건 확보 후 중단

5페이지 요청
   ↓
rownum <= 50
   ↓
50건 확보 후 중단
```

### 페이징 처리 ANTI 패턴

아래처럼 바깥쪽에서만 `between`으로 페이지 범위를 제한하면 쿼리는 간결해 보이지만 `STOPKEY`가 사라질 수 있습니다.

```sql
select *
from (
  select rownum no, a.*
  from (
    /* SQL Body */
  ) a
)
where no between (:page - 1) * 10 + 1 and (:page * 10);
```

```text
안쪽 rownum <= N 없음
   ↓
COUNT STOPKEY 없음
   ↓
전체 범위 처리
   ↓
앞쪽 일부만 반환해도 DB는 계속 읽음
```

책의 중요한 경고는 이 부분입니다. `ORDER BY` 아래쪽의 `ROWNUM` 조건은 단순한 조건절이 아니라 `Top N Stopkey`를 작동시키는 열쇠입니다.

### 부분범위 처리 가능한 SQL 작성

부분범위 처리 가능한 SQL은 다음 조건을 만족해야 합니다.

| 조건 | 설명 |
|---|---|
| 인덱스 사용 가능 | 조건절이 인덱스를 제대로 탈 수 있어야 합니다. |
| 정렬 생략 가능 | `ORDER BY`와 인덱스 순서가 맞아야 합니다. |
| 조인 방식 적절 | 대개 NL 조인이 유리합니다. |
| Stopkey 조건 유지 | `rownum <= N`을 안쪽에 둬야 합니다. |

---

## ✅ 이 절(5.3.2)의 핵심 통찰

1. Top N 쿼리는 상위 N건만 반환하는 쿼리입니다.
2. 인덱스 정렬 순서를 활용하면 N건을 읽는 순간 멈출 수 있습니다.
3. 실행계획의 `COUNT STOPKEY`는 조기 중단이 작동한다는 신호입니다.
4. 페이징에서는 안쪽 `rownum <= N` 조건이 반드시 필요합니다.
5. 겉보기로 간결한 `between` 페이징은 전체범위 처리를 유발할 수 있습니다.

## 🔗 흐름 속 위치

5.3.1이 인덱스로 `ORDER BY` 소트를 생략하는 원리였다면, 5.3.2는 거기에 `ROWNUM`을 결합해 N건만 읽고 멈추는 최적화를 설명합니다.

📌 한 줄 요약: Top N 쿼리의 힘은 정렬된 인덱스를 읽다가 필요한 N건에서 즉시 멈추는 `STOPKEY`에 있습니다.
