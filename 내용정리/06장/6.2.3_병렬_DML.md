## 6.2.3 병렬 DML

### 핵심 개념

병렬 DML은 여러 프로세스가 동시에 INSERT, UPDATE, DELETE, MERGE를 수행하는 방식입니다. 대량 작업 시간을 줄일 수 있지만, 리소스 사용량과 동시성 영향을 함께 고려해야 합니다.

```text
대량 DML
   ↓
작업 범위 분할
   ↓
여러 병렬 서버가 동시 처리
   ↓
전체 소요시간 단축
```

### 병렬 DML 활성화

Oracle에서는 병렬 DML을 사용하려면 세션 설정이 필요합니다.

```sql
alter session enable parallel dml;
```

이후 힌트나 객체 병렬 속성을 통해 병렬 처리를 유도할 수 있습니다.

```sql
insert /*+ parallel(t 4) */ into target t
select /*+ parallel(s 4) */ *
from source s;
```

### 장점과 비용

| 구분 | 내용 |
---|---|
| 장점 | 대량 작업 경과시간을 줄일 수 있습니다. |
| 비용 | CPU, I/O, Undo, Redo 사용량이 동시에 증가합니다. |
| 위험 | OLTP 업무와 리소스 경합이 커질 수 있습니다. |
| 적합 | 배치, 적재, 대량 정비 작업 |

병렬도는 높을수록 무조건 좋은 것이 아닙니다.

```text
병렬도 증가
   ↓
처리량 증가 가능
   ↓
동시에 I/O/CPU/로그 경합 증가
   ↓
특정 지점 이후 성능 악화 가능
```

---

## ✅ 이 절(6.2.3)의 핵심 통찰

1. 병렬 DML은 대량 작업을 여러 프로세스로 나누어 처리합니다.
2. 세션에서 병렬 DML을 활성화해야 합니다.
3. Direct Path Insert와 함께 대량 적재에 자주 활용됩니다.
4. 병렬도 증가는 리소스 경합도 함께 증가시킵니다.
5. OLTP 시간대에는 신중하게 사용해야 합니다.

## 🔗 흐름 속 위치

6.2.1과 6.2.2가 I/O 경로를 바꾸는 방식이었다면, 6.2.3은 처리 주체를 병렬화해 경과시간을 줄이는 방식입니다.

📌 한 줄 요약: 병렬 DML은 대량 작업을 빠르게 끝내는 힘이 있지만, 시스템 리소스를 한꺼번에 쓰는 비용을 동반합니다.
