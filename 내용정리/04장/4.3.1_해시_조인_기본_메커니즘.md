## 4.3.1 해시 조인 기본 메커니즘

### 핵심 개념
해시 조인은 **작은 입력으로 해시 테이블을 만들고, 큰 입력을 스캔하며 해시 탐색으로 매칭**합니다.

```text
Build Input 읽기
   ↓
조인 키로 해시 테이블 생성
   ↓
Probe Input 읽기
   ↓
해시 테이블 lookup
   ↓
매칭 행 반환
```

### 2단계 — Build / Probe (그림 4-10)

```sql
select /*+ ordered use_hash(c) */
       e.사원번호, e.사원명, e.입사일자, c.고객번호, c.고객명, c.전화번호, c.최종주문금액
from   사원 e, 고객 c
where  c.관리사원번호 = e.사원번호
and    e.입사일자 >= '19960101' and e.부서코드 = 'Z123'   -- Build Input(작은 쪽) 조건
and    c.최종주문금액 >= 20000;                            -- Probe Input(큰 쪽) 조건
```

**① Build 단계** — 작은 쪽(사원, 조건 만족분)을 읽어 **PGA의 Hash Area에 해시 테이블 생성**.
```text
사원번호를 해시 함수 f(x)에 입력 → 해시 체인에 연결
   [A] → 0003(김철수) → 0006(손수희)
   [B] → 0001(홍길동)
   [C] → 0005(이영희)
(PGA에 다 못 담으면 Temp 테이블스페이스에 저장 → 4.3.3)
```

**② Probe 단계** — 큰 쪽(고객)을 하나씩 읽어 해시 테이블 탐색.
```text
관리사원번호를 '같은 해시 함수 f(x)'에 입력 → 해당 해시 체인만 스캔 → 같은 사원번호 찾으면 조인 성공
(Probe = outer 루프, 큰 쪽을 순차 스캔)
```

| 단계 | 설명 |
|---|---|
| **Build** | **작은 쪽**을 PGA Hash Area에 해시 테이블로 구성 |
| **Probe** | **큰 쪽**을 읽으며 같은 해시 함수로 탐색 |
| Join | 같은 해시 체인의 키 비교 후 결과 생성 |

### CS 대응
HashMap join입니다. 작은 집합을 `HashMap<JoinKey, Rows>`로 만들고 큰 집합을 순회하며 `get(key)`(해시 체인 = Separate Chaining)를 수행합니다. Build와 Probe에 **동일한 해시 함수**를 써야 같은 키가 같은 버킷에 들어갑니다.

---

## ✅ 이 절(4.3.1)의 핵심 통찰
1. 해시 조인은 등치 조인에 강합니다.
2. Build Input은 가능한 작아야 합니다.
3. 인덱스 반복 탐색 대신 메모리 해시 탐색을 사용합니다.

📌 한 줄 요약: 해시 조인은 작은 쪽을 HashMap으로 만들고 큰 쪽을 스캔하며 매칭하는 조인입니다.
