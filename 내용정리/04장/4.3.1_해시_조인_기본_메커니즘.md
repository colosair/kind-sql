## 4.3.1 해시 조인 기본 메커니즘

### 핵심 개념
해시 조인은 **작은 입력으로 해시 테이블을 만들고, 큰 입력을 스캔하며 해시 탐색으로 매칭**합니다.

```text
Build Input 읽기
   ↓
조인 키로 해시 테이블 생성
   ↓
Probe Input 읽기
   ↓
해시 테이블 lookup
   ↓
매칭 행 반환
```

| 단계 | 설명 |
|---|---|
| Build | 작은 쪽을 PGA에 해시 테이블로 구성 |
| Probe | 큰 쪽을 읽으며 해시 탐색 |
| Join | 같은 해시 버킷의 키 비교 후 결과 생성 |

### CS 대응
HashMap join입니다. 작은 집합을 `HashMap<JoinKey, Rows>`로 만들고 큰 집합을 순회하며 `containsKey`를 수행합니다.

---

## ✅ 이 절(4.3.1)의 핵심 통찰
1. 해시 조인은 등치 조인에 강합니다.
2. Build Input은 가능한 작아야 합니다.
3. 인덱스 반복 탐색 대신 메모리 해시 탐색을 사용합니다.

📌 한 줄 요약: 해시 조인은 작은 쪽을 HashMap으로 만들고 큰 쪽을 스캔하며 매칭하는 조인입니다.
