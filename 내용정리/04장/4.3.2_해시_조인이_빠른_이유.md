## 4.3.2 해시 조인이 빠른 이유

### 핵심 개념
해시 조인이 빠른 이유는 **대량 조인을 랜덤 인덱스 반복 탐색이 아니라 메모리 해시 탐색으로 바꾸기 때문**입니다.

```text
NL 조인
Outer N건 × Inner 인덱스 탐색

해시 조인
Build 1회 + Probe 1회
```

| 비교 | NL 조인 | 해시 조인 |
|---|---|---|
| 핵심 비용 | 반복 탐색 | 양쪽 입력 스캔 + 해시 |
| 자료구조 | B*Tree 인덱스 | Hash Table |
| 적합 | 소량 OLTP | 대량 등치 조인 |

### reduce I/O 관점
해시 조인은 조인 판단을 인덱스/테이블 반복 방문에서 PGA 내 해시 테이블 조회로 옮깁니다. 비싼 블록 반복 접근을 싼 메모리 lookup으로 전환합니다.

---

## ✅ 이 절(4.3.2)의 핵심 통찰
1. 해시 조인은 반복 랜덤 액세스를 줄입니다.
2. 대량 등치 조인에서 NL보다 유리한 경우가 많습니다.
3. PGA에 Build Input이 잘 들어갈수록 빠릅니다.

📌 한 줄 요약: 해시 조인은 조인 탐색을 B*Tree 반복 탐색에서 HashMap lookup으로 바꿉니다.
