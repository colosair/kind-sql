## 4.3.4 해시 조인 실행계획 제어

### 핵심 개념
해시 조인 제어의 핵심은 **해시 조인 사용 여부와 Build/Probe 입력 방향**입니다.

| 힌트 | 의미 |
|---|---|
| `USE_HASH` | 해시 조인 유도 |
| `LEADING` | 조인 순서 지정 |
| `SWAP_JOIN_INPUTS` | Build/Probe 방향 전환 유도 |
| `NO_SWAP_JOIN_INPUTS` | 입력 방향 유지 유도 |

### 예시
```sql
SELECT /*+ LEADING(c o) USE_HASH(o) */
       *
FROM 고객 c, 주문 o
WHERE o.고객번호 = c.고객번호;
```

```text
HASH JOIN
  TABLE ACCESS FULL 고객   -- Build 후보
  TABLE ACCESS FULL 주문   -- Probe 후보
```

### 분석 포인트
작은 쪽이 Build Input이 되어야 PGA 사용량이 줄고, 해시 테이블 lookup 효율이 좋아집니다.

---

## ✅ 이 절(4.3.4)의 핵심 통찰
1. 해시 조인은 어떤 입력으로 해시 테이블을 만들지가 중요합니다.
2. 작은 입력을 Build로 잡아야 메모리와 TEMP 부담이 줄어듭니다.
3. 힌트는 조인 순서와 Build 방향을 함께 확인해야 합니다.

📌 한 줄 요약: 해시 조인 제어는 작은 쪽으로 HashMap을 만들게 하는 일입니다.
