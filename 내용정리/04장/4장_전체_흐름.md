## 4장 조인 튜닝 전체 흐름

### 전체 축
4장의 핵심은 조인을 **같은 문제를 서로 다른 자료구조로 푸는 방식**으로 이해하는 것입니다.

```text
조인 = 두 집합에서 매칭 행 찾기
   ↓
NL 조인: Outer마다 B*Tree/테이블 반복 탐색
   ↓
소트 머지 조인: 정렬 후 투 포인터 병합
   ↓
해시 조인: 작은 쪽 HashMap 생성 후 Probe
   ↓
서브쿼리 변환: 문법을 효율적인 조인 형태로 변환
```

## 4.1 NL 조인

### 핵심 개념
NL 조인은 소량 Outer와 좋은 Inner 인덱스가 만날 때 강합니다.

| 절 | 핵심 |
|---|---|
| 4.1.1 | Outer 행마다 Inner 반복 탐색 |
| 4.1.2 | 조인 순서와 `USE_NL` 제어 |
| 4.1.3 | 반복 수행 과정 분석 |
| 4.1.4 | Outer 축소와 Inner 인덱스 효율화 |
| 4.1.5 | 소량 OLTP에 적합 |
| 4.1.6 | `Starts`와 반복 액세스 확인 |
| 4.1.7 | Batching/Prefetch로 랜덤 I/O 완화 |

```text
Outer Row 수 × Inner 접근 비용
   ↓
NL 조인 전체 비용
```

## 4.2 소트 머지 조인

### 핵심 개념
소트 머지 조인은 정렬 비용을 지불하고, 이후 순차 병합으로 조인을 수행합니다.

| 절 | 핵심 |
|---|---|
| 4.2.1 | SGA는 공유 캐시, PGA는 작업공간 |
| 4.2.2 | 양쪽 정렬 후 병합 |
| 4.2.3 | 랜덤 반복 대신 순차 처리 |
| 4.2.4 | 대량/비등치/정렬 활용 조인 |
| 4.2.5 | `USE_MERGE`로 제어 |
| 4.2.6 | Sort 비용과 PGA/TEMP 의존 |

```text
Sort A + Sort B
   ↓
Merge Join
   ↓
순차 처리
```

## 4.3 해시 조인

### 핵심 개념
해시 조인은 작은 입력을 HashMap으로 만들고 큰 입력을 스캔하며 lookup합니다.

| 절 | 핵심 |
|---|---|
| 4.3.1 | Build/Probe 구조 |
| 4.3.2 | 반복 B*Tree 탐색을 HashMap lookup으로 전환 |
| 4.3.3 | Build가 메모리를 넘으면 TEMP I/O 발생 |
| 4.3.4 | `USE_HASH`, Build 방향 제어 |
| 4.3.5 | 조인 메소드 선택 기준 |

```text
작은 입력 Build
   ↓
Hash Table 생성
   ↓
큰 입력 Probe
```

## 4.4 서브쿼리 조인

### 핵심 개념
서브쿼리는 문법이고, 옵티마이저는 이를 조인 형태로 변환해 실행할 수 있습니다.

| 절 | 핵심 |
|---|---|
| 4.4.1 | 서브쿼리 변환 필요성 |
| 4.4.2 | EXISTS/IN과 세미/안티 조인 |
| 4.4.3 | 뷰 경계와 조건 이동 |
| 4.4.4 | 스칼라 서브쿼리 = NL 조인 + 캐싱(입력 값 NDV 적을 때 이득) |

```text
Subquery / View / Scalar Subquery
   ↓
Unnesting / View Merging / Predicate Pushing
   ↓
효율적인 조인 실행계획
```

## ✅ 4장의 핵심 통찰
1. 조인의 본질은 두 집합에서 매칭 행을 찾는 것입니다.
2. NL 조인은 B*Tree 반복 탐색, 소트 머지는 정렬 배열 병합, 해시는 HashMap lookup입니다.
3. NL 조인은 첫 행 응답과 소량 OLTP에 강하지만 반복 랜덤 액세스에 취약합니다.
4. 소트 머지 조인은 정렬 비용을 지불하고 대량 순차 병합으로 이득을 봅니다.
5. 해시 조인은 대량 등치 조인에서 반복 탐색을 메모리 lookup으로 바꿉니다.
6. 서브쿼리와 뷰는 문법 경계가 아니라 최적화 변환 가능성을 기준으로 봐야 합니다.
7. 스칼라 서브쿼리는 NL 조인 + 캐싱 구조이며, 입력 값 종류가 적을 때만 캐싱이 이득입니다.
8. 조인 튜닝도 결국 `reduce I/O`입니다. 비싼 반복 블록 접근을 더 싼 자료구조 탐색으로 옮기는 작업입니다.

## 🔗 흐름 속 위치
3장은 단일 테이블 액세스와 인덱스 효율을 다뤘습니다. 4장은 그 원리를 여러 테이블로 확장합니다. Inner 인덱스 효율, 랜덤 액세스, 정렬 생략, 메모리 사용이라는 앞선 개념들이 조인 메소드 선택 기준으로 다시 등장합니다.
