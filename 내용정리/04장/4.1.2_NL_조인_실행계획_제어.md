## 4.1.2 NL 조인 실행계획 제어

### 핵심 개념
NL 조인 실행계획 제어의 핵심은 **조인 순서와 조인 방식**을 의도대로 잡는 것입니다.

```text
어떤 테이블을 먼저 읽을 것인가
   ↓
Outer Row 수 결정
   ↓
Inner 반복 탐색 횟수 결정
   ↓
NL 조인 비용 결정
```

### 실행계획 읽기 — 위쪽이 Outer

```text
Execution Plan
0  SELECT STATEMENT
1   NESTED LOOPS
2    TABLE ACCESS (BY INDEX ROWID) OF '사원'    ← 위쪽 = Outer(Driving)
3     INDEX (RANGE SCAN) OF '사원_X1'
4    TABLE ACCESS (BY INDEX ROWID) OF '고객'    ← 아래쪽 = Inner(Driven, 반복)
5     INDEX (RANGE SCAN) OF '고객_X1'
```
"위쪽 사원 테이블 기준으로 아래쪽 고객 테이블과 NL 조인한다"로 읽습니다 — **NESTED LOOPS 바로 아래 첫 자식이 Outer, 다음 자식이 Inner**입니다. 각 테이블을 인덱스로 액세스하는 것도 실행계획에서 확인됩니다.

### 제어 힌트

| 힌트 | 의미 |
|---|---|
| `ORDERED` | FROM 절 순서대로 조인 |
| `LEADING(테이블)` | 조인 순서(Outer) 지정 |
| `USE_NL(테이블)` | 그 테이블을 NL 조인의 Inner로 |
| `INDEX(테이블 인덱스)` | 특정 인덱스 사용 유도 |

```sql
-- 사원을 Outer로, 고객을 NL Inner로
select /*+ ordered use_nl(c) */
       e.사원명, c.고객명, c.전화번호
from   사원 e, 고객 c
where  e.입사일자 >= '19960101' and c.관리사원번호 = e.사원번호;
```

---

## ✅ 이 절(4.1.2)의 핵심 통찰
1. NL 조인은 어떤 테이블이 Outer가 되는지가 매우 중요합니다.
2. 힌트는 조인 순서, 조인 방식, 인덱스 접근을 함께 맞출 때 의미가 있습니다.
3. 힌트보다 먼저 통계와 인덱스 설계가 올바른지 봐야 합니다.

📌 한 줄 요약: NL 조인 제어는 작은 쪽을 먼저 읽고 큰 쪽은 인덱스로 반복 탐색하게 만드는 일입니다.
