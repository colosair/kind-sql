## 4.2.2 소트 머지 조인 기본 메커니즘

### 핵심 개념
소트 머지 조인은 **두 입력을 조인 키 순서로 정렬한 뒤, 정렬된 순서를 따라 병합하며 조인**합니다.

```text
Input A 정렬
Input B 정렬
   ↓
두 정렬 결과를 앞에서부터 비교
   ↓
같은 조인 키끼리 매칭
```

| 단계 | 설명 |
|---|---|
| Sort | 양쪽 입력을 조인 키 기준 정렬 |
| Merge | 정렬된 두 집합을 순차 비교 |
| Join | 같은 키 구간을 매칭 |

### CS 대응
Merge Sort의 merge 단계와 같습니다. 두 정렬 배열을 투 포인터로 훑으면서 같은 키를 찾습니다.

---

## ✅ 이 절(4.2.2)의 핵심 통찰
1. 소트 머지 조인은 정렬 비용을 먼저 지불하고 순차 병합합니다.
2. 인덱스가 이미 정렬 순서를 제공하면 Sort 비용을 줄일 수 있습니다.
3. 대량 데이터를 랜덤 탐색보다 순차 처리로 바꾸는 전략입니다.

📌 한 줄 요약: 소트 머지 조인은 정렬된 두 집합을 투 포인터로 병합하는 조인입니다.
