## 4.1.1 NL 조인 기본 메커니즘

### 핵심 개념
NL 조인은 **바깥쪽 테이블에서 읽은 행마다 안쪽 테이블을 반복 탐색하는 조인 방식**입니다.

```text
Outer Row 1
   ↓
Inner 테이블/인덱스 탐색
   ↓
Outer Row 2
   ↓
Inner 테이블/인덱스 탐색
```

| 구분 | 설명 |
|---|---|
| Outer | 먼저 읽는 입력 |
| Inner | Outer 행마다 반복 탐색하는 입력 |
| 핵심 비용 | Outer 건수 × Inner 탐색 비용 |

### 실행계획
```text
NESTED LOOPS
  TABLE ACCESS BY INDEX ROWID 고객
  TABLE ACCESS BY INDEX ROWID 주문
    INDEX RANGE SCAN 주문_X1
```

### CS 대응
NL 조인은 이중 for-loop입니다. 다만 안쪽 루프가 단순 전체 스캔이 아니라 인덱스 탐색이면 `for each outer -> B*Tree lookup` 구조가 됩니다.

---

## ✅ 이 절(4.1.1)의 핵심 통찰
1. NL 조인의 성능은 Outer Row 수와 Inner 탐색 효율의 곱으로 결정됩니다.
2. Inner 쪽 조인 컬럼 인덱스가 중요합니다.
3. 소량 Outer를 빠르게 처리하는 OLTP에 잘 맞습니다.

📌 한 줄 요약: NL 조인은 작은 바깥 입력으로 안쪽 인덱스를 반복 탐색할 때 강합니다.
