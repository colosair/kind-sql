## 4.1.1 NL 조인 기본 메커니즘

### 핵심 개념 — 조인의 기본이자 통일 원리

> "조인의 기본은 **NL 조인**이다. NL 조인은 **인덱스를 이용한 조인**이다. ... 소트 머지 조인과 해시 조인도 **프로세싱 과정은 NL 조인과 다르지 않다.** 조인할 때 **어떤 자료구조를 사용하느냐의 차이일 뿐**이다."

이것이 4장 전체를 관통하는 **조인 통일 원리**입니다. 세 조인 방식 모두 "Outer를 한 건씩 읽어 Inner를 찾는 중첩 루프"이고, Inner를 찾는 자료구조만 다릅니다(NL=인덱스, 소트 머지=정렬된 양쪽, 해시=해시 테이블).

```sql
-- 1996년 1월 1일 이후 입사한 사원이 관리하는 고객 데이터 추출
select e.사원명, c.고객명, c.전화번호
from   사원 e, 고객 c
where  e.입사일자 >= '19960101'        -- Outer 조건
and    c.관리사원번호 = e.사원번호;     -- 조인 조건
```

---

### NL 조인 = 이중 for-loop

```text
〈 C / Java 〉                  〈 PL/SQL 〉
for (i=0; i<100; i++) {        for outer in 1..100 loop      -- Outer(사원)
  for (j=0; j<100; j++) {        for inner in 1..100 loop    -- Inner(고객)
    // Do Anything                  ...
  }                              end loop;
}                              end loop;
```

NL 조인은 위 중첩 루프와 **같은 수행 구조**입니다. Outer 테이블에서 읽은 행마다 Inner 테이블을 반복 탐색합니다.

| 구분 | 설명 |
|------|------|
| **Outer(Driving)** | 먼저 읽는 입력 (조건으로 적게 추리는 게 유리) |
| **Inner(Driven)** | Outer 행마다 반복 탐색하는 입력 |
| 핵심 비용 | **Outer 건수 × Inner 탐색 비용** |

---

### 수행 과정 6단계 (그림 4-3, 4-4)

`사원_X1(입사일자)`, `고객_X1(관리사원번호)` 인덱스가 있을 때:

```text
① 사원_X1에서 입사일자 >= '19960101' 첫 레코드 찾기        ┐
② ROWID로 사원 테이블 레코드 액세스                        │ Outer 한 건
③ 사원번호 '0006'으로 고객_X1 인덱스 탐색 (조인!)          ┐
④ ROWID로 고객 테이블 레코드 액세스                        │ Inner 반복
⑤ 고객_X1을 한 건 더 스캔, 관리사원번호 '0006' 확인        │
⑥ ROWID로 고객 테이블 레코드 액세스 ... (Inner 끝나면 ①로) ┘
```

실행계획:
```text
NESTED LOOPS
  TABLE ACCESS BY INDEX ROWID 사원         ← Outer
    INDEX RANGE SCAN 사원_X1
  TABLE ACCESS BY INDEX ROWID 고객         ← Inner (Outer 건수만큼 반복)
    INDEX RANGE SCAN 고객_X1
```

### CS 대응

NL 조인은 **이중 for-loop**이며, 안쪽 루프가 전체 스캔이 아니라 **인덱스 탐색(B*Tree lookup)** 이면 `for each outer → O(log n) lookup` 구조가 됩니다. Inner를 Recursive하게 반복 수행하는 것은 아니고(각주), 데이터 액세스 순서를 보여주는 의사 코드입니다.

---

## ✅ 이 절(4.1.1)의 핵심 통찰

1. **조인 통일 원리**: NL·소트 머지·해시 조인은 모두 중첩 루프이며, **Inner를 찾는 자료구조만** 다르다.
2. NL 조인은 **인덱스를 이용한 조인** = 이중 for-loop.
3. 성능은 **Outer 건수 × Inner 탐색 비용** — Outer를 조건으로 적게 추리고 Inner 조인 컬럼에 인덱스가 있어야 한다.
4. 수행은 "Outer 한 건 → Inner 탐색 반복"의 6단계로, 데이터 액세스 순서가 실행계획의 들여쓰기로 드러난다.
5. 소량 Outer를 빠르게 처리하는 **OLTP에 잘 맞는다**.

## 🔗 흐름 속 위치

3장까지의 인덱스 튜닝이 단일 테이블 액세스였다면, 4장은 그 원리를 **여러 테이블 조인**으로 확장합니다. NL 조인이 그 기본이며, 4.1.2(제어)·4.1.3(과정 분석)·4.1.4(튜닝 포인트)로 이어집니다. 소트 머지(4.2)·해시(4.3)는 같은 중첩 루프에 다른 자료구조를 쓴 변형입니다.

📌 한 줄 요약: NL 조인은 인덱스를 이용한 이중 for-loop이며, 모든 조인의 기본 — Outer를 적게 추리고 Inner 인덱스를 잘 타게 하는 것이 핵심입니다.
