## 4.1.6 NL 조인 튜닝 실습

### 핵심 개념
NL 조인 튜닝 실습에서는 실행계획을 보고 **반복되는 Inner 액세스가 과도한지**를 찾아야 합니다.

```text
실행계획 확인
   ↓
Outer Row 추정/실측 확인
   ↓
Inner 인덱스 스캔량 확인
   ↓
테이블 액세스 횟수 확인
   ↓
조인 순서/인덱스/조건 재설계
```

| 증상 | 원인 후보 | 개선 |
|---|---|---|
| Inner 반복이 많음 | Outer가 큼 | 조인 순서 변경 |
| Inner Range Scan 넓음 | 인덱스 컬럼 순서 부적절 | 결합 인덱스 조정 |
| 테이블 방문 많음 | 인덱스 필터 부족 | 컬럼 추가/Covered 검토 |

### 실습 — 불필요한 테이블 액세스 찾기

```text
Rows  Row Source Operation
   5  NESTED LOOPS
   3   TABLE ACCESS BY INDEX ROWID OF 사원
2780    INDEX RANGE SCAN OF 사원_X1          ← 인덱스 2,780건 스캔
   5   TABLE ACCESS BY INDEX ROWID OF 고객
   8    INDEX RANGE SCAN OF 고객_X1
```
**진단**: 사원_X1을 2,780건 스캔해 사원 테이블을 2,780번 액세스했는데, 부서코드='2123' 필터 결과는 **3건**뿐입니다. → **2,777번이 헛걸음**(테이블 액세스 후 필터). 인덱스에 **테이블 필터 조건(부서코드)을 추가**하면 됩니다(3.1.4).

```text
개선: 사원_X1에 부서코드 추가 → 인덱스에서 필터링
   3   TABLE ACCESS BY INDEX ROWID OF 사원
   3    INDEX RANGE SCAN OF 사원_X1          ← 2780 → 3 (불필요한 테이블 액세스 사라짐)
```

### 실행계획 체크 포인트

| 증상 | 원인 후보 | 개선 |
|------|------|------|
| `Starts`/Rows가 큰 Inner | Outer가 큼 | 조인 순서 변경 |
| 인덱스 스캔 ≫ 테이블 결과 | 테이블 필터 비중 큼 | 인덱스에 필터 컬럼 추가 |
| 테이블 방문 많음 | 인덱스 필터 부족 | 컬럼 추가/Covered 검토 |

---

## ✅ 이 절(4.1.6)의 핵심 통찰
1. NL 조인은 `Starts` 또는 반복 실행 횟수를 반드시 봐야 합니다.
2. Inner 단계의 작은 비용도 반복되면 전체 병목이 됩니다.
3. 튜닝은 조인 순서와 Inner 인덱스 설계를 함께 다룹니다.

📌 한 줄 요약: NL 조인 실습의 핵심은 반복되는 안쪽 액세스 비용을 숫자로 확인하는 것입니다.
