## 4.4.3 뷰와 조인

### 핵심 개념
뷰와 조인 튜닝의 핵심은 **뷰 경계가 조건 이동과 조인 순서 선택을 막는지**를 확인하는 것입니다.

```text
뷰 내부 SQL
   ↓
외부 조건과 분리
   ↓
먼저 큰 집합 생성 가능
   ↓
이후 필터링
```

| 최적화 | 설명 |
|---|---|
| View Merging | 뷰를 외부 SQL과 합쳐 최적화 |
| Predicate Pushing | 외부 조건을 뷰 내부로 밀어넣음 |
| Materialization | 뷰 결과를 먼저 생성 |

### 예시
```sql
SELECT *
FROM (SELECT * FROM 주문 WHERE 주문일자 >= :d1) v
WHERE v.고객번호 = :cust_no;
```

고객번호 조건이 뷰 내부로 들어가면 더 적은 주문만 읽을 수 있습니다.

---

## ✅ 이 절(4.4.3)의 핵심 통찰
1. 뷰는 SQL을 보기 좋게 만들지만 최적화 경계를 만들 수 있습니다.
2. 조건이 뷰 내부로 들어가야 읽는 양이 줄어듭니다.
3. View Merging과 Predicate Pushing 여부를 실행계획에서 확인해야 합니다.

📌 한 줄 요약: 뷰 튜닝은 조건과 조인 순서가 뷰 경계에 갇히지 않게 하는 일입니다.
