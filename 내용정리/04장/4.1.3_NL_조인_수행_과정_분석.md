## 4.1.3 NL 조인 수행 과정 분석

### 핵심 개념
NL 조인은 실행계획을 위에서 아래로만 읽지 말고, **반복 구조**로 해석해야 합니다.

```text
1. Outer 테이블에서 조건 만족 행 추출
2. 각 Outer 행의 조인 키 획득
3. Inner 인덱스 Range/Unique Scan
4. Inner 테이블 ROWID 액세스
5. 조인 결과 반환
```

| 분석 포인트 | 질문 |
|---|---|
| Outer 건수 | 반복 횟수가 많은가 |
| Inner 인덱스 | 조인 키로 정확히 접근하는가 |
| Inner 테이블 액세스 | 랜덤 액세스가 과도한가 |
| 필터 위치 | 조인 후에야 걸러지는가 |

### 조건절 비교 순서 — 어느 조건이 언제 평가되나

NL 조인 분석의 핵심은 **각 조건절이 수행 과정의 어느 단계에서 평가되는가**입니다.
```sql
select /*+ ordered use_nl(c) index(e) index(c) */
       e.사원번호, e.사원명, e.입사일자, c.고객번호, c.전화번호, c.최종주문금액
from   사원 e, 고객 c
where  c.관리사원번호 = e.사원번호    -- ① 조인 조건
and    e.입사일자 >= '19960101'       -- ② Outer(사원) 액세스/필터
and    e.부서코드 = '2123'            -- ③ Outer(사원) 필터
and    c.최종주문금액 >= 20000;       -- ④ Inner(고객) 필터
```
```text
수행 순서:
  사원_X1(입사일자)로 ② 액세스 → 사원 테이블에서 ③ 부서코드 필터  ┐ Outer
     ↓ 살아남은 사원마다                                          │
  ① 사원번호로 고객_X1 탐색(조인) → 고객 테이블에서 ④ 필터        ┘ Inner 반복
```
- ②는 Outer 인덱스 **액세스 조건**, ③은 Outer 테이블 **필터**.
- ①은 **조인 조건**(Inner 인덱스 액세스), ④는 Inner 테이블 **필터**.
- ③·④ 필터가 약하면(많이 통과) Inner 반복·랜덤 액세스가 그만큼 누적됩니다.

### 실행계획 해독
```text
| Id | Operation                    | Name   | Starts | A-Rows |
|  1 |  NESTED LOOPS                |        |   1    |        |
|  2 |   TABLE ACCESS BY INDEX ROWID| 사원   |   1    |   3    |   ← Outer 3건
|  3 |    INDEX RANGE SCAN          | 사원_X1|   1    |        |
|  4 |   TABLE ACCESS BY INDEX ROWID| 고객   |   3    |   5    |   ← Starts=3 (Outer만큼 반복!)
|  5 |    INDEX RANGE SCAN          | 고객_X1|   3    |        |
```
**`Starts`가 Outer 건수(3)와 같으면 NL 조인 정상** — Inner가 Outer 건수만큼 반복됐다는 뜻(부록.4의 핵심). Outer가 많아질수록 Inner의 작은 비효율이 증폭됩니다.

---

## ✅ 이 절(4.1.3)의 핵심 통찰
1. NL 조인은 Inner 접근이 Outer 건수만큼 반복됩니다.
2. 실행계획의 각 단계 비용보다 반복 횟수를 함께 봐야 합니다.
3. Inner 테이블 랜덤 액세스가 누적되면 급격히 느려집니다.

📌 한 줄 요약: NL 조인 분석은 “안쪽 탐색이 몇 번 반복되는가”를 세는 일입니다.
