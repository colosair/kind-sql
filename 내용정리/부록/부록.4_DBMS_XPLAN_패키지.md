## 부록.4 DBMS_XPLAN 패키지

### 핵심 개념
`DBMS_XPLAN`은 Oracle 실행계획을 다양한 형식으로 출력하는 패키지입니다. 특히 실제 실행 통계와 Predicate 정보를 함께 보면 튜닝 진단력이 크게 올라갑니다.

```text
SQL 실행
   ↓
Cursor에 실행계획과 통계 누적
   ↓
DBMS_XPLAN.DISPLAY_CURSOR로 출력
   ↓
예상 Rows vs 실제 Rows, Predicate 정보 확인
```

### ⚠️ A-Rows를 보려면 먼저 수행 통계 수집을 켜야 한다

`ALLSTATS LAST`는 단계별 **실제** 수행 통계(A-Rows 등)를 보여주지만, 그냥 실행하면 통계가 수집되지 않아 A-Rows가 비어 나옵니다. 둘 중 하나를 먼저 해야 합니다.

```sql
-- 방법 1: 분석 대상 SQL에 힌트 (가장 간편)
select /*+ gather_plan_statistics */ ...

-- 방법 2: 세션 레벨 설정
alter session set statistics_level = all;
```

그 후 바로 직전 수행한 SQL의 단계별 통계를 조회합니다.

```sql
SELECT *
FROM TABLE(
  DBMS_XPLAN.DISPLAY_CURSOR(NULL, NULL, 'ALLSTATS LAST')
);
```

> 참고: `DISPLAY_CURSOR`가 권한 오류가 나면 `v_$session`, `v_$sql`, `v_$sql_plan`에 대한 SELECT 권한이 필요합니다(DBA에게 요청).

출력되는 핵심 컬럼은 다음과 같습니다.

| 컬럼 | 의미 | 연결 |
|---|---|---|
| `Starts` | 해당 단계 **반복 수행 횟수** | NL 조인 Inner 반복([[4.1.6_NL_조인_튜닝_실습]]) |
| `E-Rows` | 옵티마이저 예상 행 수 | 7장 카디널리티 |
| `A-Rows` | 실제 행 수 | E-Rows와 비교 |
| `A-Time` | 실제 누적 수행 시간 | 병목 단계 식별 |
| `Buffers` | 단계별 논리적 I/O | `reduce I/O` 검증 |

### format 옵션

| 옵션 | 의미 |
|---|---|
| `ALLSTATS LAST` | 마지막 실행의 실제 행 수와 통계 |
| `+PREDICATE` | Access/Filter Predicate 표시 |
| `+PEEKED_BINDS` | Peeked Bind 값 표시 |
| `+OUTLINE` | 실행계획 재현 힌트 정보 |

(EXPLAIN PLAN 결과는 `DBMS_XPLAN.DISPLAY`, 실제 수행한 커서는 `DISPLAY_CURSOR`로 봅니다.)

### 가장 중요한 해석 포인트
```text
E-Rows
   ↓
옵티마이저 예상 행 수

A-Rows
   ↓
실제 실행 행 수

둘의 차이가 큼
   ↓
통계정보 / 선택도 / 히스토그램 / 바인드 문제 의심
```

### Predicate 해석
```text
access(...)
   ↓
인덱스 스캔 범위를 줄인 조건

filter(...)
   ↓
읽은 뒤 걸러낸 조건
```

이 구분은 3장의 액세스 조건과 필터 조건을 실제 실행계획에서 확인하는 핵심 방법입니다.

---

## ✅ 이 항목의 핵심 통찰
1. `DBMS_XPLAN`은 실행계획을 가장 실전적으로 읽게 해주는 핵심 도구입니다.
2. A-Rows를 보려면 **먼저** `/*+ gather_plan_statistics */` 힌트나 `statistics_level=all`이 필요합니다(없으면 비어 나옴).
3. `E-Rows`와 `A-Rows` 차이는 옵티마이저 추정 오류의 신호입니다.
4. `Starts`는 단계별 반복 횟수로, NL 조인 Inner 반복 진단의 핵심입니다.
5. Predicate 정보를 보면 조건이 액세스인지 필터인지 확인할 수 있습니다.
6. `ALLSTATS LAST +PREDICATE`는 튜닝 진단의 기본 조합입니다.

📌 한 줄 요약: DBMS_XPLAN은 실행계획을 추정치가 아니라 실제 Rows와 Predicate까지 포함한 진단 자료로 바꿔줍니다.
