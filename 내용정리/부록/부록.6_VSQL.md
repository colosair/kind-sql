## 부록.6 V$SQL

### 핵심 개념
`V$SQL`은 Shared Pool에 올라온 SQL의 수행 통계를 조회하는 동적 성능 뷰입니다. SQL별 실행 횟수, CPU, elapsed time, buffer gets 등을 보고 튜닝 대상을 찾을 수 있습니다.

```text
시스템에서 SQL 실행
   ↓
Shared Pool에 커서 정보 저장
   ↓
V$SQL에 누적 통계 기록
   ↓
비용이 큰 SQL 후보 추출
```

| 컬럼 | 의미 | 튜닝 관점 |
|---|---|---|
| `SQL_ID` | SQL 식별자 | 상세 분석의 키 |
| `EXECUTIONS` | 실행 횟수 | 반복 비용 판단 |
| `BUFFER_GETS` | 논리적 I/O 누적 | 핵심 선별 지표 |
| `DISK_READS` | 물리적 I/O 누적 | 디스크 읽기 부담 |
| `CPU_TIME` | CPU 사용 시간 | CPU 병목 |
| `ELAPSED_TIME` | 전체 경과 시간 | 사용자 체감 비용 |
| `ROWS_PROCESSED` | 처리 행 수 | 건당 비용 계산 |

### 예시
```sql
SELECT sql_id,
       executions,
       buffer_gets,
       disk_reads,
       elapsed_time,
       rows_processed
FROM v$sql
ORDER BY buffer_gets DESC;
```

### 해석 포인트
```text
총 BUFFER_GETS가 큼
   ↓
시스템 전체 I/O 부담 큼

EXECUTIONS가 많음
   ↓
건당 비용이 작아도 누적 부담 가능

BUFFER_GETS / EXECUTIONS가 큼
   ↓
한 번 실행할 때 비싼 SQL
```

---

## ✅ 이 항목의 핵심 통찰
1. `V$SQL`은 튜닝 대상 SQL을 찾는 출발점입니다.
2. 총 비용과 실행당 비용을 나누어 봐야 합니다.
3. `SQL_ID`를 확보한 뒤 `DBMS_XPLAN.DISPLAY_CURSOR`로 상세 실행계획을 확인할 수 있습니다.

📌 한 줄 요약: V$SQL은 시스템 안에서 어떤 SQL이 I/O와 시간을 많이 쓰는지 찾는 목록표입니다.
