## 부록.5 실시간 SQL 모니터링

### 핵심 개념
실시간 SQL 모니터링은 오래 걸리거나 병렬로 수행되는 SQL의 진행 상황을 **실시간으로 추적**하는 기능입니다.

```text
장시간 SQL 실행
   ↓
실시간 모니터링 정보 수집
   ↓
각 실행계획 단계별 진행률/시간/I/O 확인
   ↓
현재 병목 단계 파악
```

| 확인 항목 | 의미 |
|---|---|
| Plan Line | 실행계획 단계 |
| Activity | 시간 소비 위치 |
| Rows | 단계별 처리 행 수 |
| I/O | 읽기/쓰기 발생량 |
| Temp | 정렬·해시 spill 여부 |
| Parallel | 병렬 실행 분포 |

### 유용한 상황
| 상황 | 이유 |
|---|---|
| 장시간 배치 SQL | 완료 전 병목 확인 가능 |
| 병렬 쿼리 | PX 서버별 작업 분포 확인 |
| Hash/Sort 대량 작업 | TEMP 사용과 spill 확인 |
| 잘못된 조인 순서 | 특정 단계에서 폭발하는 Rows 확인 |

### 해석 흐름
```text
현재 오래 머무는 Plan Line 확인
   ↓
그 단계의 Rows/I/O/TEMP 확인
   ↓
조인, 정렬, Full Scan, 파티션 접근 중 병목 판별
```

---

## ✅ 이 항목의 핵심 통찰
1. 실시간 SQL 모니터링은 끝난 뒤가 아니라 실행 중인 SQL을 관찰합니다.
2. 긴 SQL에서 어느 실행계획 단계가 병목인지 바로 볼 수 있습니다.
3. Hash Join, Sort, Parallel Query, TEMP 사용 분석에 특히 유용합니다.

📌 한 줄 요약: 실시간 SQL 모니터링은 오래 도는 SQL의 병목 위치를 실행 중에 보여주는 도구입니다.
