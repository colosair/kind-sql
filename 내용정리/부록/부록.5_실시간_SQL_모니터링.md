## 부록.5 실시간 SQL 모니터링

### 핵심 개념

**11g부터 제공**되는 실시간 SQL 모니터링(Real-Time SQL Monitoring)은 오래 걸리거나 병렬로 수행되는 SQL의 진행 상황을 **실행 중에 실시간으로 추적**하는 기능입니다. (부록.1~4가 "끝난 SQL"을 분석했다면, 이건 "지금 도는 SQL"을 본다는 점이 다릅니다.)

### 모니터링 대상

오라클이 실시간으로 모니터링하는 대상은 자동으로 선정됩니다.

```text
1. CPU time 또는 I/O time을 5초 이상 소비한 SQL   (※ _sqlmon_threshold 파라미터로 조정)
2. 병렬(Parallel) SQL
3. monitor 힌트를 지정한 SQL
```

> 단, 실행계획이 **300 라인을 넘으면 모니터링 대상에서 제외**됩니다(`_sqlmon_max_planlines` 파라미터를 300 이상으로 설정하면 회피).

### 수집 정보

| 뷰 | 내용 |
|------|------|
| `V$SQL_MONITOR` | SQL 단위 모니터링 정보 |
| `V$SQL_PLAN_MONITOR` | 실행계획 **단계별** 모니터링 정보 |

> SQL을 실행하는 동안 **1초마다 자동 갱신**됩니다. 수행을 마친 후에도 적어도 **1분간** 정보를 유지하지만, 이후 캐시 공간이 부족하면 언제든 밀려날 수 있습니다.

---

### 더 쉬운 방법 — report_sql_monitor 함수

뷰를 직접 조회하는 대신 `dbms_sqltune.report_sql_monitor` 함수를 씁니다.

```sql
-- 텍스트 리포트
SQL> select dbms_sqltune.report_sql_monitor(sql_id=>'6x50yqwz81sfa') from dual;

-- HTML 리포트 (그래프·진행률 시각화)
SQL> select dbms_sqltune.report_sql_monitor(sql_id=>'6x50yqwz81sfa', type=>'html') from dual;
```

이 함수는 **I/O 발생량과 수행시간을 전체 + 실행계획 단계별로 구분**해 보여줍니다. 리포트는 크게 세 부분입니다.

| 리포트 섹션 | 내용 |
|------|------|
| **Global Stats** | Elapsed/CPU/IO time, Buffer Gets, Read/Write Reqs·Bytes 등 전체 통계 |
| **Parallel Execution Details** | QC·각 PX 서버(Set 1/Set 2)별 일량 분포 |
| **SQL Plan Monitoring Details** | 단계별 Rows(Estim/Actual), Cost, Start Active, Execs, Read/Write, Mem/Temp |

### 병렬 쿼리 분석에 특히 유용

> 모든 병렬 프로세스의 실행계획 단계별 일량을 **집계해서 보여줄 뿐 아니라, 병렬 프로세스별 일량을 따로 요약(Parallel Execution Details)** 해서 보여줍니다.

이 덕분에 **병렬 작업 불균형(skew)** — 특정 PX 서버에 일이 몰려 전체가 느려지는 상황 — 을 한눈에 진단할 수 있습니다. (6.2.3 병렬 DML의 작동 확인과 직결)

---

### 해석 흐름

```text
오래 머무는 Plan Line(단계) 확인
   ↓
그 단계의 Rows(Actual) / Read·Write / Temp 확인
   ↓
조인·정렬·Full Scan·파티션 접근 중 병목 판별
   ↓ (병렬이면)
PX 서버 간 일량이 고른가? (skew 확인)
```

---

## ✅ 이 항목의 핵심 통찰

1. 실시간 SQL 모니터링(11g~)은 **실행 중인 SQL**을 1초 단위로 관찰한다 — 끝난 뒤 분석하는 부록.1~4와 다르다.
2. 대상은 **CPU/IO 5초 이상·병렬 SQL·monitor 힌트** SQL이며, 실행계획 300라인 초과 시 제외된다.
3. `V$SQL_MONITOR`/`V$SQL_PLAN_MONITOR`에 수집되며, 수행 후 최소 1분 유지된다.
4. **`dbms_sqltune.report_sql_monitor`(특히 `type=>'html'`)** 로 전체·단계별 통계를 시각적으로 본다.
5. **병렬 쿼리의 PX 서버별 일량 분포**를 보여줘 병렬 불균형(skew) 진단에 특히 유용하다.

## 🔗 흐름 속 위치

부록.4(DBMS_XPLAN)가 완료된 SQL의 실측 통계였다면, 부록.5는 **장시간·병렬 SQL을 실행 중에** 관찰합니다. 6.2.3 병렬 DML의 "PX COORDINATOR 위/아래" 작동 확인을 실제 일량 분포로 보완합니다. 다음 부록.6(V$SQL)은 캐시에 남은 SQL들의 누적 통계를 조회하는 방법입니다.

📌 한 줄 요약: 실시간 SQL 모니터링은 오래 도는·병렬 SQL의 병목 위치와 PX 서버별 일량 분포를 **실행 중에** 보여주는 도구이며, `report_sql_monitor(...,type=>'html')`이 가장 편리합니다.
