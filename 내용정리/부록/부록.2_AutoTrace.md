## 부록.2 AutoTrace

### 핵심 개념
AutoTrace는 SQL 실행 후 **실행계획과 실행 통계**를 간단히 확인하는 도구입니다.

```text
SQL 실행
   ↓
Execution Plan 출력
   ↓
Statistics 출력
   ↓
논리적 I/O, 물리적 I/O, 정렬, 네트워크 왕복 등 확인
```

| 항목 | 의미 | 튜닝 관점 |
|---|---|---|
| consistent gets | 논리적 I/O | 가장 핵심 지표 |
| physical reads | 물리적 I/O | 캐시 미스/디스크 읽기 |
| sorts | 정렬 발생 여부 | 5장 소트 튜닝 연결 |
| rows processed | 처리 행 수 | 카디널리티 검증 |

### 사용 예
```sql
SET AUTOTRACE ON

SELECT *
FROM 주문
WHERE 고객번호 = :cust_no;
```

### 해석 포인트
```text
실행계획 확인
   ↓
Index Range Scan 여부 확인
   ↓
Statistics에서 consistent gets 확인
   ↓
인덱스를 탔는데도 logical I/O가 큰지 판단
```

AutoTrace는 빠른 1차 진단에 좋습니다. 다만 복잡한 SQL의 실제 실행 단계별 통계까지 세밀히 보려면 SQL Trace나 DBMS_XPLAN의 실행 통계가 더 적합합니다.

---

## ✅ 이 항목의 핵심 통찰
1. AutoTrace는 실행계획과 기본 실행 통계를 함께 보여주는 간단한 진단 도구입니다.
2. 튜닝에서 가장 먼저 볼 지표는 대개 `consistent gets`입니다.
3. 실행계획이 좋아 보여도 논리적 I/O가 크면 비효율이 남아 있는 것입니다.

📌 한 줄 요약: AutoTrace는 SQL의 실행경로와 전체 I/O 비용을 빠르게 보는 1차 진단 도구입니다.
