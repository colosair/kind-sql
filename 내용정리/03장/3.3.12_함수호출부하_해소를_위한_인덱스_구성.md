## 3.3.12 함수호출부하 해소를 위한 인덱스 구성

### 핵심 개념
함수호출부하는 **많은 행을 읽은 뒤 행마다 함수를 실행할 때** 크게 발생합니다. 인덱스 구성으로 함수 호출 대상을 줄이거나, 함수 결과를 인덱스화할 수 있습니다.

```text
대량 후보 스캔
   ↓
행마다 함수 호출
   ↓
CPU 부하 + I/O 낭비

인덱스에서 후보 축소
   ↓
함수 호출 횟수 감소
```

| 방식 | 설명 |
|---|---|
| 조건 컬럼 인덱스 보강 | 함수 호출 전 후보를 줄임 |
| 함수 기반 인덱스 | 함수 결과 자체를 인덱스에 저장 |
| 계산 결과 컬럼화 | 자주 쓰는 변환 결과를 미리 저장 |

### 예시
```sql
WHERE SUBSTR(전화번호, 1, 3) = '010'
```

일반 인덱스에서는 컬럼 가공 때문에 인덱스 활용이 어려울 수 있습니다. 함수 기반 인덱스를 만들면 함수 결과 기준으로 탐색할 수 있습니다.

---

## ✅ 이 절(3.3.12)의 핵심 통찰
1. 함수 호출 비용은 후보 행 수에 비례해 커집니다.
2. 인덱스는 I/O뿐 아니라 반복 CPU 연산도 줄이는 도구가 됩니다.
3. 컬럼 가공 조건은 함수 기반 인덱스나 데이터 모델 정비로 풀 수 있습니다.

📌 한 줄 요약: 함수호출부하 튜닝은 계산을 매번 하지 않도록 후보를 줄이거나 계산 결과를 인덱스화하는 일입니다.
