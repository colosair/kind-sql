## 3.2.2 부분범위 처리 구현

### 핵심 개념
부분범위 처리는 DB, 애플리케이션, 화면이 모두 **필요한 만큼만 가져오는 방식**으로 협력해야 구현됩니다.

```text
인덱스 순서로 읽기
   ↓
FETCH 크기만큼 반환
   ↓
화면에 표시
   ↓
다음 요청 시 이어서 처리
```

| 레이어 | 역할 |
|---|---|
| DB | 정렬된 순서로 빠르게 일부 행 반환 |
| WAS/Driver | fetch size 단위로 가져옴 |
| 화면 | 페이지 또는 스크롤 단위로 소비 |

### 예시
```sql
SELECT *
FROM 주문
WHERE 고객번호 = :cust_no
ORDER BY 주문일자 DESC;
```

```text
인덱스: (고객번호, 주문일자 DESC)
```

이 구성이면 고객별 최신 주문부터 읽을 수 있으므로 첫 화면 응답이 빨라집니다.

---

## ✅ 이 절(3.2.2)의 핵심 통찰
1. 부분범위 처리는 SQL만의 문제가 아니라 fetch 방식과 화면 사용 패턴까지 포함합니다.
2. ORDER BY를 만족하는 인덱스가 있으면 정렬 부담 없이 앞부분을 반환할 수 있습니다.
3. 전체 결과를 요구하는 연산이 끼면 부분범위 처리 이점이 사라질 수 있습니다.

📌 한 줄 요약: 부분범위 처리는 인덱스 순서, fetch 단위, 화면 소비 방식이 맞아야 효과가 납니다.
