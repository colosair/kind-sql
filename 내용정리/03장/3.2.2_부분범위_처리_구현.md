## 3.2.2 부분범위 처리 구현

### 핵심 개념 — 구현은 '클라이언트 프로그램'의 몫

부분범위 처리로 구현할지 말지는 **클라이언트 프로그램을 개발하는 개발자의 몫**입니다. 같은 SQL이라도 클라이언트가 결과를 어떻게 소비하느냐에 따라 전체범위/부분범위가 갈립니다.

| 도구 | 구현 방식 |
|------|------|
| 토드(Toad), 오렌지(Orange) | **부분범위 처리** (앞부분만 먼저 보여줌) |
| SQL*Plus, SQL Server 쿼리 분석기 | 전체범위 처리 |

---

### ① 전체범위 처리 (AllRange) — 끝까지 읽음

```java
public static void execute(Connection con) throws Exception {
  int arraysize = 10;
  Statement stmt = con.createStatement();
  stmt.setFetchSize(arraysize);
  ResultSet rs = stmt.executeQuery("select object_id, object_name from all_objects");
  while (rs.next()) {                              // ⚠️ 끝까지 전부 읽음
    System.out.println(rs.getLong(1) + " : " + rs.getString(2));
  }
  rs.close(); stmt.close();
}
```
arraysize를 10으로 줘도 `while(rs.next())`로 **결과를 끝까지** 읽으므로 부분범위 효과가 없습니다.

### ② 부분범위 처리 (PartialRange) — arraysize만큼 읽고 멈춤

```java
// arraysize만큼 읽고 멈추는 fetch()
public static int fetch(ResultSet rs, int arraysize) throws Exception {
  int i = 0;
  while (rs.next()) {
    System.out.println(rs.getLong(1) + " : " + rs.getString(2));
    if (++i == arraysize) return i;               // ★ arraysize 채우면 멈춤
  }
  return i;
}
public static void execute(Connection con) throws Exception {
  int arraysize = 10;
  ...
  while (true) {
    int r = fetch(rs, arraysize);                 // arraysize만큼만 Fetch
    if (r < arraysize) break;
    System.out.print("Enter to Continue ... (Q)uit? ");   // ★ 사용자 요청 시 다음 Fetch
    String input = in.readLine();
    if (input.equals("Q")) break;
  }
  rs.close(); stmt.close();
}
```
arraysize만큼 읽고 멈췄다가, **사용자 요청(Enter)이 있을 때만 다시 Fetch**합니다.

> 개발자가 매번 이렇게 일일이 구현할 순 없고, **개발 프레임워크에 미리 구현된 기능(페이징/스크롤)** 을 활용합니다.

---

### 인덱스가 받쳐줘야 한다

부분범위 처리가 효과를 내려면 **ORDER BY를 만족하는 인덱스**가 있어야 정렬 없이 앞부분을 바로 반환할 수 있습니다(2.2.4).

```sql
select * from 주문 where 고객번호 = :cust_no order by 주문일자 desc;
-- 인덱스 (고객번호, 주문일자) → 최신 주문부터 정렬 없이 반환 → 첫 화면 즉시
```
반대로 `order by created`인데 created가 인덱스에 없으면, DB 서버는 **모든 데이터를 읽어 정렬을 마쳐야**(전체범위) 첫 행을 줄 수 있습니다.

### Array Size 조정으로 Fetch Call 최소화

- **대량을 모두 받을 때**(파일 내려받기 등): arraysize를 **크게** → Fetch Call 횟수 감소.
- **앞 일부만 보고 멈출 때**: arraysize를 **작게** → 불필요하게 많이 받아 버리는 낭비 방지. (arraysize=1,000인데 100건만 쓰고 끝나면 900건을 헛되이 전송)

---

## ✅ 이 절(3.2.2)의 핵심 통찰

1. 부분범위 처리 구현은 **클라이언트 프로그램 개발자의 몫** — 토드/오렌지는 부분범위, SQL*Plus는 전체범위다.
2. `while(rs.next())`로 끝까지 읽으면(**AllRange**) 부분범위 효과가 없다. **arraysize만큼 읽고 멈췄다 요청 시 재개**(PartialRange)해야 한다.
3. 보통은 직접 구현하지 않고 **개발 프레임워크의 페이징/스크롤** 기능을 쓴다.
4. **ORDER BY를 만족하는 인덱스**가 있어야 정렬 없이 앞부분을 즉시 반환할 수 있다.
5. Array Size는 **대량 수신엔 크게, 앞부분만이면 작게** 조정해 Fetch Call/전송 낭비를 최적화한다.
