## 3.3.7 Index Skip Scan 활용

### 핵심 개념
Index Skip Scan은 **선두 컬럼 조건이 없어도 선두 컬럼의 가능한 값을 건너뛰며 후행 컬럼을 탐색하는 방식**입니다.

```text
인덱스: (성별, 고객번호)
조건: 고객번호 = :v
   ↓
성별='M' 구간에서 고객번호 탐색
성별='F' 구간에서 고객번호 탐색
   ↓
선두 컬럼을 건너뛴 것처럼 처리
```

| 조건 | Skip Scan 적합성 |
|---|---|
| 선두 컬럼 NDV 작음 | 유리 |
| 후행 컬럼 선택도 좋음 | 유리 |
| 선두 컬럼 NDV 큼 | 불리 |

### 구체 예제 — 월별고객별판매집계

`IDX1 = (판매구분 + 판매월)`. 판매구분 NDV는 2(A/B). 판매월 between 조건만 자주 쓰이고 **판매구분은 옵션**이라고 합시다.
```sql
select count(*) from 월별고객별판매집계
where 판매월 between '201801' and '201812';     -- 선두(판매구분) 조건이 없음
```
선두 컬럼(판매구분)이 빠졌지만 NDV가 2뿐이라, **Index Skip Scan**으로 `판매구분='A'` 구간과 `'B'` 구간 각각에서 판매월 범위를 탐색합니다. 별도 인덱스를 만들지 않고 **기존 IDX1을 재활용**하는 것입니다.

```text
INDEX SKIP SCAN 월별고객별판매집계_IDX1
  → 판매구분='A' 구간에서 판매월 201801~201812 탐색
  → 판매구분='B' 구간에서 판매월 201801~201812 탐색
```
(판매구분 NDV가 2처럼 작아서 가능 — 수십·수백이면 Full Scan과 비슷해져 불리, 2.3.4)

### CS 대응
복합 정렬 배열 `(A, B)`에서 A 조건이 없더라도, A의 가능한 값이 몇 개 안 되면 `A=a1` 구간, `A=a2` 구간을 각각 탐색하는 방식입니다. Branch and Bound처럼 탐색 공간을 분할해 불필요한 큰 스캔을 피합니다.

---

## ✅ 이 절(3.3.7)의 핵심 통찰
1. Skip Scan은 선두 컬럼 조건 부재를 완전히 해결하는 마법은 아닙니다.
2. 선두 컬럼 종류가 적고 후행 조건이 강할 때 효과가 있습니다.
3. 별도 인덱스를 만들지 않고 기존 결합 인덱스를 재활용하는 선택지가 됩니다.

📌 한 줄 요약: Skip Scan은 선두 컬럼 값을 몇 개의 가상 구간으로 나눠 후행 컬럼을 반복 탐색하는 방식입니다.
