## 3.3.3 액세스 조건과 필터 조건

### 핵심 개념 — 조건절은 세 가지 역할로 나뉜다 (그림 3-30)

인덱스 스캔 효율성을 이해하려면, 조건절이 실행 시 하는 역할을 **세 가지**로 구분해야 합니다.

```text
        ① 인덱스 액세스 조건  (스캔 범위 결정)
              ╱╲
   ┌─────────────────────┐  인덱스 리프
   │ ││ │  ││  │ │ │ ││   │
   └──┬──┬────┬──────────┘
      ▼  ▼    ▼  ② 인덱스 필터 조건 (테이블에 갈지 결정)
   ┌─────────────────────┐  테이블
      ③ 테이블 필터 조건 (다음 단계/최종 결과 포함 여부)
```

| 구분 | 작동 위치 | 역할 |
|------|------|------|
| **① 인덱스 액세스 조건** | 인덱스 수직 탐색 + 리프 스캔 | **스캔 범위(시작점·종료점)를 결정** |
| **② 인덱스 필터 조건** | 인덱스 리프 스캔 중 | **테이블로 액세스할지** 결정(테이블 안 가게 거름) |
| **③ 테이블 필터 조건** | 테이블 액세스 후 | 다음 단계로 전달하거나 **최종 결과집합 포함 여부** 결정 |

> "인덱스를 이용하든, 테이블을 Full Scan 하든, **테이블 액세스 단계에서 처리되는 조건절은 모두 필터 조건**(③)이다."

---

### 같은 조건도 인덱스 구성에 따라 역할이 달라진다

2.1.5에서 본 `(성별 + 고객명)` 인덱스 예처럼, **조건절에 있다고 다 같은 역할이 아닙니다.**

```text
인덱스 (C1, C2, C3)  /  조건: C1=? AND C2=? AND C3=? AND C4=?
   C1, C2, C3 → 인덱스 액세스 조건 (스캔 범위 결정)
   C4         → 인덱스에 없으면 테이블 필터 조건
```

- 선두부터 **연속으로 `=` 조건**인 컬럼들이 인덱스 액세스 조건이 됩니다.
- 중간에 **범위 조건이나 누락**이 나오면, 그 뒤 컬럼들은 액세스 조건에서 **필터 조건으로 전락**합니다(3.3.4~3.3.5).

```text
인덱스 (A, B, C),  조건: A=? AND B BETWEEN ? AND ? AND C=?
   A → 액세스 조건
   B → 액세스 조건(범위, 여기서 스캔 멈춤 결정)
   C → B가 범위라서 액세스 조건이 못 됨 → 필터 조건  ⚠️
```

---

### 왜 이 구분이 중요한가

- **액세스 조건**은 **읽는 양(I/O)** 자체를 줄입니다.
- **필터 조건**은 읽은 뒤 버리므로 **결과 건수는 줄이지만 I/O는 덜 줄입니다.**

따라서 튜닝의 핵심은 **중요한 조건을 필터에서 액세스 조건으로 끌어올리는 것**(컬럼 순서 조정·인덱스 컬럼 추가)입니다. 단, **인덱스 필터 조건(②)** 으로라도 거르면 적어도 **테이블 랜덤 액세스는 줄일 수 있습니다**(3.1.4 — SAL 컬럼 추가가 이 경우).

| 조건 종류 | 줄이는 것 |
|------|------|
| ① 인덱스 액세스 조건 | **인덱스 스캔량** (가장 강력) |
| ② 인덱스 필터 조건 | **테이블 랜덤 액세스** (스캔량은 그대로) |
| ③ 테이블 필터 조건 | 최종 결과 건수만 (I/O 절감 거의 없음) |

---

## ✅ 이 절(3.3.3)의 핵심 통찰

1. 조건절은 **① 인덱스 액세스 / ② 인덱스 필터 / ③ 테이블 필터** 세 역할로 나뉜다(그림 3-30).
2. **① 액세스 조건만 인덱스 스캔량을 줄인다.** ②는 테이블 액세스를, ③은 결과 건수만 줄인다.
3. 선두부터 연속 `=` 컬럼이 액세스 조건이며, **범위 조건·누락 뒤의 컬럼은 필터 조건으로 전락**한다.
4. 튜닝의 핵심은 중요한 조건을 **필터 → 액세스로 끌어올리는 것**(컬럼 순서·추가).
5. 액세스로 못 올려도 **인덱스 필터(②)** 로 거르면 테이블 랜덤 액세스는 줄일 수 있다(3.1.4).

## 🔗 흐름 속 위치

3.3 인덱스 스캔 효율화의 **이론적 토대**입니다. 7.1.3 비용 공식의 "유효 인덱스 선택도(액세스 조건만) vs 유효 테이블 선택도(액세스+필터)" 구분이 바로 이것입니다. 이어지는 3.3.4~3.3.5는 "범위 조건·비등치 선행 컬럼이 어떻게 후행을 필터로 전락시키는가"를, 3.3.6~3.3.7은 그 회피책을 다룹니다.

📌 한 줄 요약: 조건절은 **인덱스 액세스(스캔량↓)·인덱스 필터(테이블 액세스↓)·테이블 필터(결과만↓)** 세 역할로 나뉘며, 튜닝은 중요한 조건을 필터에서 액세스로 끌어올리는 것입니다.
