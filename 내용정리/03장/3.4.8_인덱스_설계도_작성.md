## 3.4.8 인덱스 설계도 작성

### 핵심 개념
인덱스 설계도는 **테이블별 SQL 패턴, 조건 컬럼, 정렬 컬럼, 기존 인덱스를 한눈에 놓고 조정하는 문서**입니다.

```text
SQL 목록 수집
   ↓
조건절/조인절/정렬절 추출
   ↓
컬럼 사용 패턴 그룹화
   ↓
인덱스 후보 설계
   ↓
중복/누락 검토
```

| 항목 | 기록 내용 |
|---|---|
| SQL ID/업무 | 어떤 화면/배치에서 쓰는가 |
| 조건 컬럼 | 등치/범위/옵션 여부 |
| 정렬 컬럼 | ORDER BY/GROUP BY |
| 조회 컬럼 | Covered 가능성 |
| 실행 빈도 | 중요도 판단 |
| 기존 인덱스 | 중복/대체 가능성 |

### 설계 관점
인덱스 설계도는 개별 SQL만 보는 시야를 workload 전체로 넓혀줍니다. 비슷한 SQL을 묶고, 하나의 인덱스로 여러 SQL을 지원할지, 핵심 SQL에 전용 인덱스를 줄지 판단할 수 있습니다.

---

## ✅ 이 절(3.4.8)의 핵심 통찰
1. 인덱스 설계도는 인덱스 튜닝을 감이 아니라 구조적 의사결정으로 바꿉니다.
2. 조건, 정렬, 출력, 빈도를 함께 보아야 실제 비용 구조가 보입니다.
3. 좋은 설계도는 중복 인덱스를 줄이고 핵심 SQL의 실행경로를 안정화합니다.

📌 한 줄 요약: 인덱스 설계도는 SQL 묶음 전체의 I/O 비용을 줄이기 위한 지도입니다.
