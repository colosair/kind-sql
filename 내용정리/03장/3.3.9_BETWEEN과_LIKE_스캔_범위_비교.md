## 3.3.9 BETWEEN과 LIKE 스캔 범위 비교

### 핵심 개념 — LIKE보다 BETWEEN

월별 데이터(2019년 1~12월)를 조회할 때 흔히 `LIKE`를 씁니다.
```sql
where 판매월 like '2019%'                        -- 편해서 자주 씀
where 판매월 between '201901' and '201912'        -- 더 정확한 표현
```
둘 다 범위검색 조건이라 **앞서 본 비효율 원리(후행 컬럼이 필터로 전락)가 똑같이 적용**됩니다. 다만 데이터 분포·조건절 값에 따라 인덱스 스캔량이 달라질 수 있습니다.

> **결론: LIKE보다 BETWEEN이 낫다.** 코딩은 LIKE가 편하지만, **BETWEEN을 쓰면 적어도 손해는 안 봅니다.** (LIKE는 경계가 모호할 수 있고, `'2019%'`처럼 자릿수가 안 맞으면 의도와 다른 범위를 스캔할 수 있음)

예: 인덱스 `(판매월 + 판매구분)`, 판매구분 'A'90%/'B'10%.
```sql
-- 조건절 1: where 판매월 BETWEEN '201901' and '201912' and 판매구분 = 'B'
-- 조건절 2: where 판매월 LIKE '2019%'                  and 판매구분 = 'B'
```
판매월이 **선행 범위 조건**이라 두 경우 모두 판매구분='B'는 **필터로 전락**합니다 — 판매월 2019 구간 전체(A 90% 포함)를 읽고 B만 거릅니다.

```text
BETWEEN 'A' AND 'B'  →  'A' 이상 'B' 이하 (경계 명확)
LIKE 'A%'            →  'A'로 시작하는 범위 (사실상 BETWEEN과 유사하나 경계 모호 가능)
LIKE '%A'            →  시작점 없음 → Range Scan 불가 (2.2.2)
```

| 조건 | 스캔 가능성 | 주의점 |
|---|---|---|
| `BETWEEN a AND b` | 가능 | 양 끝 경계 포함 |
| `LIKE 'ABC%'` | 가능 | prefix 범위 |
| `LIKE '%ABC'` | 어려움 | 시작점 없음 |

### 핵심 차이
`LIKE 'ABC%'`는 시작 문자열이 고정되어 있어 인덱스 시작점을 잡을 수 있습니다. 반면 앞에 와일드카드가 붙으면 어디서 시작해야 할지 알 수 없어 일반적인 Range Scan이 어렵습니다.

---

## ✅ 이 절(3.3.9)의 핵심 통찰
1. LIKE도 prefix가 고정되면 인덱스 Range Scan이 가능합니다.
2. 앞쪽 와일드카드는 인덱스 정렬 순서를 활용하기 어렵게 만듭니다.
3. 문자열 범위 조건은 경계값 포함 여부와 정렬 규칙을 함께 봐야 합니다.

📌 한 줄 요약: 인덱스는 왼쪽부터 정렬되므로 LIKE도 왼쪽 prefix가 고정될 때만 효율적입니다.
