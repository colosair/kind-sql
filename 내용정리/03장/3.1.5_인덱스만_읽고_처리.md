## 3.1.5 인덱스만 읽고 처리

### 핵심 개념
인덱스만 읽고 처리한다는 것은 **쿼리에 필요한 컬럼이 모두 인덱스에 있어 테이블을 방문하지 않는 상태**입니다.

```text
조건 컬럼 + 출력 컬럼이 모두 인덱스에 존재
   ↓
인덱스 리프 블록만 스캔
   ↓
TABLE ACCESS BY INDEX ROWID 생략
   ↓
랜덤 액세스 제거
```

| 방식 | 테이블 방문 | 특징 |
|---|---:|---|
| 일반 인덱스 조회 | 있음 | ROWID로 테이블 액세스 |
| Covered 조회 | 없음 | 인덱스만으로 결과 생성 |

### 실행계획
```sql
SELECT 고객번호, 주문일자
FROM 주문
WHERE 고객번호 = :cust_no;
```

```text
INDEX RANGE SCAN 주문_X1
```

테이블 액세스 단계가 없다면 인덱스만 읽고 처리한 것입니다.

### CS 대응
HashMap의 value 전체를 가져오지 않고, key-value index 구조 안에 필요한 metadata가 이미 들어 있어 lookup 결과만으로 처리가 끝나는 구조와 비슷합니다.

---

## ✅ 이 절(3.1.5)의 핵심 통찰
1. 테이블 랜덤 액세스를 완전히 제거하는 가장 강한 방법은 인덱스만 읽고 처리하는 것입니다.
2. 출력 컬럼까지 인덱스에 포함하면 Covered Query가 가능합니다.
3. 단, 인덱스 비대화와 DML 부하 증가를 함께 고려해야 합니다.

📌 한 줄 요약: 가장 빠른 테이블 액세스는 테이블에 아예 가지 않는 것입니다.
