## 3.3.6 BETWEEN을 IN-List로 전환

### 핵심 개념
`BETWEEN`을 `IN-List`로 바꾸는 목적은 **넓은 범위 스캔을 여러 개의 좁은 등치 탐색으로 분해**하는 데 있습니다.

```text
A BETWEEN 1 AND 3 AND B = 'X'
   ↓
A 범위 전체 스캔

A IN (1,2,3) AND B = 'X'
   ↓
(A=1,B='X'), (A=2,B='X'), (A=3,B='X') 반복 탐색
```

| 방식 | 장점 | 단점 |
|---|---|---|
| BETWEEN | SQL 간결, 범위 처리 자연스러움 | 후행 컬럼 활용 약화 |
| IN-List | 후행 컬럼까지 좁은 탐색 가능 | IN 값이 많으면 반복 탐색 비용 증가 |

### 구체 예제 — 매물아파트매매

인덱스 `(인터넷매물 + 아파트시세코드 + 평형 + 평형타입)`. 범위검색 컬럼(인터넷매물)이 **선두**라, `인터넷매물 between '1' and '3'` 이면 중간값 `'2'` 구간에서 후행 조건과 무관하게 **전체 레코드를 다 읽어야** 합니다. (인덱스 순서를 바꾸면 좋지만 운영 시스템에선 어려움)

```sql
-- BETWEEN을 IN-List로
select 해당층, 평당가, 입력일, 해당동, ...
from   매물아파트매매
where  인터넷매물 in ('1', '2', '3')        -- BETWEEN '1' AND '3' → IN-List
and    아파트시세코드 = 'A01011350900056'
and    평형 = '59'
and    평형타입 = 'A'
order by 입력일 desc;
```
```text
Execution Plan
1   INLIST ITERATOR                         ← 값별로 반복
2    TABLE ACCESS BY INDEX ROWID  매물아파트매매
3     INDEX RANGE SCAN            매물아파트매매_PK
```

옵티마이저가 IN-List를 **`INLIST ITERATOR`로 값별 반복 탐색**(인터넷매물='1','2','3' 각각 수직 탐색 3번)으로 처리하므로, 각 탐색에서 **후행 컬럼(아파트시세코드·평형·평형타입)까지 모두 `=` 액세스 조건**으로 활용해 정확한 위치만 읽습니다(그림 3-34).

---

## ✅ 이 절(3.3.6)의 핵심 통찰
1. BETWEEN은 연속 범위라 후행 컬럼의 효율을 떨어뜨릴 수 있습니다.
2. IN-List는 작은 범위라면 등치 조건 반복으로 바꿔 스캔량을 줄일 수 있습니다.
3. IN 값이 너무 많으면 반복 탐색 자체가 부담이 되므로 손익을 봐야 합니다.

📌 한 줄 요약: BETWEEN을 IN-List로 바꾸는 것은 큰 범위 하나를 작은 정확 탐색 여러 번으로 쪼개는 전략입니다.
