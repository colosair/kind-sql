## 3.1.6 인덱스 구조 테이블

### 핵심 개념
인덱스 구조 테이블(IOT)은 **테이블 자체를 인덱스 구조로 저장하는 방식**입니다.

```text
일반 테이블
인덱스 → ROWID → 테이블

IOT
인덱스 구조 자체에 행 저장
```

| 구분 | 일반 테이블 | IOT |
|---|---|---|
| 저장 기준 | 힙 구조 | 기본키 순서 |
| 인덱스 접근 후 | 테이블 ROWID 액세스 | 리프에서 데이터 획득 |
| 유리한 경우 | 범용 DML | PK 기반 조회, 범위 조회 |

IOT는 PK 순서로 데이터가 모여 있어 PK 기반 조회와 범위 조회에 강합니다. 반면 저장 구조가 인덱스이므로 입력 패턴, 데이터 변경, 보조 인덱스 관리 비용을 고려해야 합니다.

### CS 대응
일반 테이블은 객체 배열과 별도 인덱스 맵을 둔 구조이고, IOT는 B*Tree 리프 노드에 실제 레코드 payload를 함께 저장하는 구조입니다.

---

## ✅ 이 절(3.1.6)의 핵심 통찰
1. IOT는 테이블과 인덱스의 이중 방문을 줄이기 위한 물리 설계입니다.
2. PK 중심 액세스가 많을수록 효과가 큽니다.
3. 모든 테이블에 적합한 구조는 아니며 액세스 패턴이 선명해야 합니다.

📌 한 줄 요약: IOT는 인덱스 리프를 곧 테이블 데이터 저장소로 만들어 랜덤 액세스를 줄이는 구조입니다.
