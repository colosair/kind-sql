## 7.2.3 옵티마이저에 영향을 미치는 요소

### 핵심 개념
옵티마이저는 SQL 문장만 보고 판단하지 않습니다. **통계정보, 인덱스, 파라미터, 힌트, 바인드 변수, 시스템 통계**가 함께 실행계획에 영향을 줍니다.

```text
SQL 텍스트
   ↓
객체 통계 / 시스템 통계 / 파라미터 / 힌트 / 바인드
   ↓
비용 계산
   ↓
실행계획 선택
```

| 요소 | 영향 |
|---|---|
| 테이블/인덱스 통계 | 카디널리티와 I/O 비용 |
| 히스토그램 | skew 값 선택도 |
| 인덱스 구성 | 가능한 액세스 경로 |
| 초기화 파라미터 | 비용 모델과 변환 방식 |
| 힌트 | 실행계획 선택 유도 |
| 바인드 변수 | 값별 선택도 추정 문제 |

### 정밀화
옵티마이저가 이상한 계획을 골랐다고 해서 항상 옵티마이저 자체가 문제인 것은 아닙니다. 입력 정보가 부정확하거나, SQL 표현이 변환을 막거나, 인덱스 선택지가 부적절한 경우가 많습니다.

---

## ✅ 이 절(7.2.3)의 핵심 통찰
1. 실행계획은 SQL 문장과 주변 메타데이터의 합작품입니다.
2. 통계정보와 인덱스 설계는 옵티마이저 선택지를 결정합니다.
3. 힌트는 마지막 수단에 가깝고, 먼저 입력 정보와 구조를 점검해야 합니다.

📌 한 줄 요약: 옵티마이저는 SQL만이 아니라 통계, 인덱스, 파라미터, 바인드까지 함께 보고 계획을 고릅니다.
