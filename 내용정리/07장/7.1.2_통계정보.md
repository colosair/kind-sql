## 7.1.2 통계정보

### 핵심 개념
통계정보는 옵티마이저가 비용을 계산하기 위해 사용하는 **데이터 분포 요약 정보**입니다. 크게 두 종류로 나뉩니다.

```text
통계정보
├─ 오브젝트 통계
│   ├─ 테이블 통계 (Rows, Blocks ...)
│   ├─ 인덱스 통계 (BLEVEL, LEAF_BLOCKS, CLUSTERING_FACTOR ...)
│   └─ 컬럼 통계 (NDV, 히스토그램, DENSITY, Min/Max ...)
└─ 시스템 통계 (CPU 속도, Single/Multiblock I/O 속도 등)
```

오브젝트 통계는 `dbms_stats.gather_table_stats(...)` 등으로 수집하며, 7.1.3 비용 공식의 항(BLEVEL·LEAF_BLOCKS·CLUSTERING_FACTOR·NDV)이 모두 여기서 나옵니다.

| 통계 항목 | 의미 | 실행계획 영향 |
|---|---|---|
| Rows | 테이블 행 수 | 전체 규모 판단 |
| Blocks | 테이블 블록 수 | Full Scan 비용 |
| NDV | 컬럼 값 종류 수 | 선택도 계산 |
| Histogram | 값 분포 정보 | skew 데이터 추정 |
| Clustering Factor | 인덱스 순서와 테이블 순서 유사도 | 인덱스 테이블 액세스 비용 |

### 왜 중요한가
통계정보가 오래되었거나 데이터 분포를 제대로 표현하지 못하면 옵티마이저는 잘못된 세계관에서 비용을 계산합니다.

```text
실제: 조건 결과 10건
통계 추정: 조건 결과 100만 건
   ↓
인덱스 대신 Full Scan 선택 가능
```

### CS 대응
통계정보는 쿼리 실행 전에 만든 메타데이터 기반 heuristic입니다. 데이터 전체를 매번 직접 세지 않고, 샘플·요약 정보를 이용해 비용을 추정합니다.

---

## ✅ 이 절(7.1.2)의 핵심 통찰
1. 옵티마이저는 실제 데이터를 직접 다 읽어보고 계획을 세우지 않습니다.
2. 통계정보는 비용 계산의 입력값입니다.
3. 데이터 skew가 있으면 히스토그램 같은 분포 정보가 중요합니다.
4. 클러스터링 팩터는 3장의 테이블 랜덤 액세스 비용 추정과 직접 연결됩니다.

📌 한 줄 요약: 통계정보는 옵티마이저가 실행 전에 데이터 세계를 추정하는 지도입니다.
