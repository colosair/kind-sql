## 7.2.5 개발자의 역할

### 핵심 개념
개발자의 역할은 옵티마이저가 좋은 실행계획을 선택하도록 **SQL, 인덱스, 통계, 데이터 모델을 정돈하는 것**입니다.

```text
업무 요구사항 이해
   ↓
올바른 SQL 표현
   ↓
적절한 인덱스 설계
   ↓
정확한 통계정보 유지
   ↓
실행계획 검증
```

| 역할 | 설명 |
|---|---|
| SQL 작성 | 컬럼 가공, 옵션 조건, 불필요한 변환 방해 제거 |
| 인덱스 설계 | 액세스 조건과 정렬 조건 지원 |
| 통계 관리 | 실제 데이터 분포 반영 |
| 실행계획 해석 | 예상과 실제 차이 확인 |
| 힌트 사용 | 필요한 경우 계획 안정화 |

### 앞 장과의 연결
개발자는 2~6장에서 배운 원리를 옵티마이저가 활용할 수 있게 만들어야 합니다.

```text
인덱스 컬럼 순서
   ↓
가능한 액세스 경로 제공

조인 조건 명확화
   ↓
조인 변환/순서 선택 가능

소트 생략 인덱스
   ↓
정렬 비용 감소 후보 제공
```

---

## ✅ 이 절(7.2.5)의 핵심 통찰
1. 개발자는 옵티마이저 대신 모든 계획을 손으로 고르는 사람이 아닙니다.
2. 개발자는 옵티마이저가 올바르게 판단할 수 있는 입력과 선택지를 제공해야 합니다.
3. 힌트보다 중요한 것은 SQL 구조, 인덱스, 통계정보입니다.
4. 실행계획의 추정 Rows와 실제 Rows 차이를 보는 습관이 중요합니다.

📌 한 줄 요약: 개발자의 역할은 옵티마이저를 억누르는 것이 아니라, 옵티마이저가 좋은 선택을 하도록 판을 정리하는 것입니다.
