## 7.2.6 튜닝 전문가 되는 공부방법

### 핵심 개념
튜닝 전문가는 실행계획을 외우는 사람이 아니라, **SQL이 어떤 I/O 경로와 자료구조 연산으로 실행되는지 해석하는 사람**입니다.

```text
SQL 문장
   ↓
실행계획
   ↓
인덱스/조인/정렬/해시/파티션 동작 해석
   ↓
실제 I/O와 Rows 확인
   ↓
원인 진단과 개선
```

| 학습 축 | 질문 |
|---|---|
| 실행계획 | 이 단계는 어떤 자료구조를 읽는가 |
| Rows | 예상 Rows와 실제 Rows가 맞는가 |
| I/O | 논리적 I/O가 어디서 많이 발생하는가 |
| 인덱스 | 액세스 조건인가 필터 조건인가 |
| 조인 | 반복 탐색인가, 정렬 병합인가, 해시 lookup인가 |
| 소트 | 정렬을 생략할 수 있는가 |

### 공부 방법
```text
1. 실행계획을 연산 트리로 읽기
2. 각 노드의 입력/출력 Rows 추적
3. 논리적 I/O가 큰 지점 찾기
4. 조건절이 액세스인지 필터인지 분해
5. 개선 후 실제 실행계획과 I/O 비교
```

### CS 대응
튜닝은 DB 전용 암기가 아니라 시스템 성능 분석입니다. 자료구조 선택, 메모리 locality, 캐시, 외부 정렬, 해시 파티셔닝, 비용 기반 탐색이 모두 연결됩니다.

---

## ✅ 이 절(7.2.6)의 핵심 통찰
1. 튜닝 학습의 중심은 실행계획 해석과 실제 I/O 검증입니다.
2. Rows 추정 오류를 보면 옵티마이저가 왜 잘못 판단했는지 보입니다.
3. 모든 개선은 결국 더 적은 블록을 읽거나 더 싼 레이어에서 판단하게 만드는 방향입니다.
4. SQL 튜닝은 자료구조와 시스템 성능 분석의 응용입니다.

📌 한 줄 요약: 튜닝 전문가는 실행계획을 자료구조 연산과 I/O 흐름으로 번역할 수 있는 사람입니다.
