## 1.1.3 SQL 옵티마이저

### 핵심 개념

SQL 옵티마이저는 사용자가 원하는 결과를 가장 효율적으로 얻기 위한 데이터 액세스 경로를 선택하는 DBMS 핵심 엔진입니다.

```text
SQL
   ↓
후보 실행계획 탐색
   ↓
각 실행계획 비용 계산
   ↓
최저 비용 실행계획 선택
```

> "SQL 옵티마이저는 사용자가 원하는 작업을 가장 효율적으로 수행할 수 있는 최적의 데이터 액세스 경로를 선택해 주는 DBMS의 핵심 엔진이다."

### 최적화 단계 (그림 1-3)

| 단계 | 설명 |
|---|---|
| 1. 후보군 생성 | 쿼리를 수행할 후보군이 될 만한 실행계획들을 찾아냅니다. |
| 2. 비용 산정 | 데이터 딕셔너리(Data Dictionary)에 수집된 **오브젝트 통계 + 시스템 통계**로 각 실행계획의 예상비용을 산정합니다. |
| 3. 실행계획 선택 | **최저 비용**을 나타내는 실행계획을 선택합니다. |

```text
통계정보
   ↓
비용기반 옵티마이저 ──┬──► 73
  (Decision Tree)   ├──► 128
                    ├──► (14) ◄── 최저 비용 ★ 선택
                    ├──► 64
                    ├──► 361
                    ├──► 72
                    ├──► 141
                    └──► 173
```

옵티마이저는 후보 실행계획마다 비용(73·128·14·64…)을 계산하고, 그중 **가장 싼 14**를 최종 실행계획으로 고릅니다 — 의사결정 트리(Decision Tree)에서 최저 비용 경로를 탐색하는 것과 같습니다.

### 옵티마이저는 별도 프로세스가 아니다

옵티마이저는 DBWR, LGWR 같은 백그라운드 프로세스가 아닙니다. 서버 프로세스가 가진 기능입니다.

```text
서버 프로세스
   ├─ SQL 파서
   ├─ 옵티마이저
   └─ 로우 소스 생성기
```

CS 관점에서는 옵티마이저가 탐색 공간에서 비용이 가장 낮은 경로를 찾는 `cost-based search`를 수행한다고 볼 수 있습니다.

---

## ✅ 이 절(1.1.3)의 핵심 통찰

1. 옵티마이저는 SQL의 실행경로를 선택합니다.
2. 선택 기준은 통계정보 기반 비용입니다.
3. 옵티마이저는 별도 프로세스가 아니라 서버 프로세스의 기능입니다.

## 🔗 흐름 속 위치

SQL 최적화의 주체가 바로 옵티마이저입니다. 이후 인덱스, 조인, 소트 튜닝은 모두 옵티마이저가 좋은 실행계획을 선택하게 만드는 과정입니다.

📌 한 줄 요약: 옵티마이저는 SQL 실행계획 후보를 비교해 가장 싸다고 판단한 경로를 고르는 비용 기반 탐색기입니다.
