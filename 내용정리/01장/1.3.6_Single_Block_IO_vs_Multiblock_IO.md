## 1.3.6 Single Block I/O vs. Multiblock I/O

### 핵심 개념

Single Block I/O는 한 번에 한 블록을 읽는 방식이고, Multiblock I/O는 한 번의 I/O Call로 여러 블록을 읽는 방식입니다.

```text
Single Block I/O
   ↓
I/O Call 1번에 블록 1개

Multiblock I/O
   ↓
I/O Call 1번에 블록 여러 개
```

### 비교

| 구분 | Single Block I/O | Multiblock I/O |
|---|---|---|
| 대표 작업 | 인덱스 경유 테이블 액세스 | Table Full Scan |
| I/O 단위 | 1블록 | 여러 블록 |
| 유리한 상황 | 소량 랜덤 조회 | 대량 순차 스캔 |
| 위험 | 반복되면 Call 수 폭증 | 불필요한 블록도 읽을 수 있음 |

### 인덱스와 Single Block I/O

인덱스로 ROWID를 얻어 테이블을 액세스할 때는 대개 Single Block I/O가 발생합니다.

```text
인덱스 엔트리
   ↓
ROWID
   ↓
테이블 블록 1개 읽기
   ↓
반복
```

로우 수가 많아지면 Single Block I/O가 매우 많이 반복될 수 있습니다.

### Full Scan과 Multiblock I/O

Table Full Scan은 연속된 블록을 한꺼번에 읽을 수 있으므로 대량 처리에 유리합니다.

```text
Full Scan
   ↓
연속 블록 묶음 읽기
   ↓
I/O Call 수 감소
```

> **손수레 비유 (그림 1-25)**: 미장공(서버 프로세스)이 벽돌(블록)로 담장을 쌓다 벽돌이 떨어지면 운반공이 건재상에서 벽돌을 사 옵니다. 운반공이 **손수레(Multiblock 버퍼)에 한 번에 많이 실어 오면** 미장공이 잠자는 횟수(I/O Call 대기)가 줄어 공사가 빨리 끝납니다.

한 번에 담는 블록 수는 `db_file_multiblock_read_count` 파라미터로 정합니다.

```sql
SQL> show parameter db_file_multiblock_read_count
--  NAME                            TYPE     VALUE
--  db_file_multiblock_read_count   integer  16        ← 기본값
SQL> alter session set db_file_multiblock_read_count = 128;
```

OS 레벨 I/O 단위가 1MB, 오라클 블록이 8KB이면 이 값을 **128**로 설정할 때 최대(8KB × 128 = 1MB)가 됩니다. 그 이상은 OS가 자기 단위(1MB)씩만 읽으므로 소용없습니다.

### Multiblock I/O는 익스텐트 경계를 못 넘는다

중요한 제약이 하나 있습니다. "인접한 블록"이란 **같은 익스텐트에 속한 블록**을 뜻합니다. Multiblock I/O라도 익스텐트 경계를 넘지 못합니다.

```text
한 익스텐트에 20블록, MULTIBLOCK_READ_COUNT = 8
   ↓
I/O Call: 8 + 8 + 4 (= 20)
   ↓
세 번째 Call은 4블록만 읽고 멈춤 (다음 익스텐트를 함께 읽지 않음)
```

즉, 손수레(Multiblock 버퍼)를 다 채우지 못했어도 익스텐트 벽돌이 떨어지면 거기서 끊습니다. 그래서 익스텐트가 잘게 쪼개져 있으면 Multiblock I/O의 이점이 줄어듭니다.

---

## ✅ 이 절(1.3.6)의 핵심 통찰

1. Single Block I/O는 소량 랜덤 액세스에 적합합니다.
2. Multiblock I/O는 대량 순차 스캔에 적합합니다.
3. 인덱스가 항상 빠른 것은 Single Block I/O 반복 횟수가 적을 때입니다.
4. 대량 조회에서는 Full Scan이 더 효율적일 수 있습니다.
5. Multiblock 크기는 `db_file_multiblock_read_count`로 정하되, 익스텐트 경계를 넘지 못합니다.

## 🔗 흐름 속 위치

1.3.4에서 액세스 패턴을 봤고, 이 절은 그 패턴이 실제 I/O Call 단위와 어떻게 연결되는지 설명합니다.

📌 한 줄 요약: 소량은 Single Block, 대량은 Multiblock이 유리하므로 인덱스와 Full Scan의 선택 기준이 달라집니다.
