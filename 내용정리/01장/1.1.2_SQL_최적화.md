## 1.1.2 SQL 최적화

### 핵심 개념

SQL 최적화는 사용자가 작성한 SQL을 DBMS 내부에서 실행 가능한 처리 절차로 바꾸는 과정입니다.

```text
SQL 입력
   ↓
SQL 파싱
   ↓
SQL 최적화
   ↓
로우 소스 생성
   ↓
실행
```

### SQL 파싱

SQL 파서는 SQL 문장을 분석해 파싱 트리를 만들고 문법/의미 오류를 검사합니다.

| 단계 | 설명 |
|---|---|
| 파싱 트리 생성 | SQL 구성요소를 분석해 트리 구조 생성 |
| Syntax 체크 | 키워드, 문법, 순서 오류 확인 |
| Semantic 체크 | 테이블/컬럼 존재 여부, 권한 등 의미 오류 확인 |

### SQL 최적화

옵티마이저는 통계정보를 바탕으로 여러 실행경로를 비교하고 가장 효율적이라고 판단한 실행계획을 선택합니다.

```text
후보 실행계획 생성
   ↓
통계정보 기반 비용 계산
   ↓
최저 비용 실행계획 선택
```

### 로우 소스 생성

로우 소스 생성기는 옵티마이저가 선택한 실행계획을 실제 실행 가능한 코드/프로시저 형태로 포맷팅합니다.

---

## ✅ 이 절(1.1.2)의 핵심 통찰

1. SQL 실행 전에는 파싱, 최적화, 로우 소스 생성 과정이 필요합니다.
2. 옵티마이저는 여러 후보 실행계획 중 비용이 낮은 것을 선택합니다.
3. SQL 튜닝은 이 최적화 과정에 좋은 선택지를 제공하는 작업입니다.

## 🔗 흐름 속 위치

1.1.1에서 SQL이 선언형 언어임을 봤습니다. 이 절은 선언된 SQL이 실제 실행 절차로 바뀌는 내부 과정을 설명합니다.

📌 한 줄 요약: SQL 최적화는 선언형 SQL을 실행 가능한 절차로 컴파일하는 과정입니다.
