## 2.3.6 Index Range Scan Descending

### 핵심 개념

`Index Range Scan Descending`은 인덱스 리프 블록을 역방향으로 스캔하는 방식입니다. 내림차순 정렬이나 최대값/최신값 조회에 유용합니다.

```text
인덱스 정방향
   ↓
작은 값 → 큰 값

Descending Scan
   ↓
큰 값 → 작은 값
```

### 사용 예 — 옵티마이저가 자동으로 거꾸로 읽음

```sql
select * from emp where empno > 0 order by empno desc;
```
```text
Execution Plan
1   TABLE ACCESS (BY INDEX ROWID) OF 'EMP'
2    INDEX (RANGE SCAN DESCENDING) OF 'PK_EMP'   ← 역방향 스캔, SORT 없음
```
EMPNO에 인덱스가 있으면 옵티마이저가 알아서 **인덱스를 거꾸로 읽는** 실행계획을 세웁니다. 만약 거꾸로 읽지 않는다면 **`index_desc`** 힌트로 유도할 수 있습니다.

`(고객ID, 주문일시)` 인덱스에서 `where 고객ID=:cust_id order by 주문일시 desc`도 마찬가지로, 고객ID 구간의 **오른쪽 끝에서 좌측으로** 역방향 스캔해 최신순으로 반환합니다(SORT 생략).

### MAX 값 — 뒤에서 한 건만 읽고 멈춤

MAX 값을 구할 때 해당 컬럼에 인덱스가 있으면, **인덱스를 뒤에서부터 한 건만 읽고 멈추는** 실행계획이 자동 수립됩니다(2.2.6과 동일 원리).

```sql
select deptno, dname, loc
     , (select max(sal) from emp where deptno = d.deptno)
from   dept d;
```
```text
Execution Plan
1   SORT (AGGREGATE)
2    FIRST ROW                                  ← 한 건만
3     INDEX (RANGE SCAN (MIN/MAX)) OF 'EMP_X02'
4   TABLE ACCESS (FULL) OF 'DEPT'
```
(더 자세한 내용은 5.3.3 '최소값/최대값 구하기')

### Top N과 연결

```sql
select *
from (
  select *
  from 주문
  where 고객ID = :cust_id
  order by 주문일시 desc
)
where rownum <= 10;
```

```text
Index Range Scan Descending
   ↓
최신 10건 읽음
   ↓
COUNT STOPKEY로 중단
```

---

## ✅ 이 절(2.3.6)의 핵심 통찰

1. Descending Scan은 인덱스를 역방향으로 읽습니다.
2. 내림차순 정렬을 생략할 수 있습니다.
3. 최신 이력, 최대값, Top N 조회에 유용합니다.
4. 5장의 Top N/이력 조회 튜닝과 직접 연결됩니다.

## 🔗 흐름 속 위치

2장의 마지막 절로, 인덱스 정렬성을 정방향뿐 아니라 역방향으로도 활용할 수 있음을 보여줍니다.

📌 한 줄 요약: Index Range Scan Descending은 정렬된 인덱스를 거꾸로 읽어 최신값과 내림차순 결과를 빠르게 얻는 방식입니다.
