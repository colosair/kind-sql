## 2.3.6 Index Range Scan Descending

### 핵심 개념

`Index Range Scan Descending`은 인덱스 리프 블록을 역방향으로 스캔하는 방식입니다. 내림차순 정렬이나 최대값/최신값 조회에 유용합니다.

```text
인덱스 정방향
   ↓
작은 값 → 큰 값

Descending Scan
   ↓
큰 값 → 작은 값
```

### 사용 예

```sql
select *
from 주문
where 고객ID = :cust_id
order by 주문일시 desc;
```

인덱스가 `(고객ID, 주문일시)`이면 고객ID 구간을 역방향으로 읽어 최신 주문부터 반환할 수 있습니다.

```text
고객ID 구간 찾기
   ↓
구간의 오른쪽 끝으로 이동
   ↓
역방향 스캔
   ↓
최신순 반환
```

### Top N과 연결

```sql
select *
from (
  select *
  from 주문
  where 고객ID = :cust_id
  order by 주문일시 desc
)
where rownum <= 10;
```

```text
Index Range Scan Descending
   ↓
최신 10건 읽음
   ↓
COUNT STOPKEY로 중단
```

---

## ✅ 이 절(2.3.6)의 핵심 통찰

1. Descending Scan은 인덱스를 역방향으로 읽습니다.
2. 내림차순 정렬을 생략할 수 있습니다.
3. 최신 이력, 최대값, Top N 조회에 유용합니다.
4. 5장의 Top N/이력 조회 튜닝과 직접 연결됩니다.

## 🔗 흐름 속 위치

2장의 마지막 절로, 인덱스 정렬성을 정방향뿐 아니라 역방향으로도 활용할 수 있음을 보여줍니다.

📌 한 줄 요약: Index Range Scan Descending은 정렬된 인덱스를 거꾸로 읽어 최신값과 내림차순 결과를 빠르게 얻는 방식입니다.
