## 2장 인덱스 기본 전체 흐름

### 핵심 개념

2장은 인덱스를 **정렬된 B*Tree 자료구조**로 이해하는 장입니다. 인덱스는 수직 탐색으로 시작 지점을 찾고, 수평 탐색으로 필요한 범위를 읽으며, ROWID로 테이블을 액세스합니다.

```text
조건절
   ↓
B*Tree 수직 탐색
   ↓
리프 시작 지점 도달
   ↓
수평 범위 스캔
   ↓
ROWID로 테이블 액세스
```

| 절 | 핵심 질문 | 핵심 답 |
|---|---|---|
| 2.1.1 | 인덱스 튜닝은 무엇인가 | 읽을 블록 범위를 줄이는 일입니다. |
| 2.1.2 | 인덱스는 어떻게 생겼는가 | 루트, 브랜치, 리프의 B*Tree 구조입니다. |
| 2.1.3 | 수직 탐색은 무엇인가 | 시작 리프 지점을 찾는 과정입니다. |
| 2.1.4 | 수평 탐색은 무엇인가 | 리프 블록 범위를 읽는 과정입니다. |
| 2.1.5 | 결합 인덱스는 어떻게 정렬되는가 | 선두 컬럼부터 차례로 정렬됩니다. |
| 2.2.1 | 인덱스를 사용한다는 것은 무엇인가 | 스캔 범위와 테이블 액세스를 줄이는 것입니다. |
| 2.2.2 | 왜 Range Scan이 안 되는가 | 컬럼 가공, 선두 컬럼 누락 등으로 정렬성을 못 씁니다. |
| 2.2.3 | 가장 중요한 조건은 무엇인가 | 선두 컬럼 조건입니다. |
| 2.2.4 | 인덱스로 소트를 생략할 수 있는가 | ORDER BY와 인덱스 순서가 맞으면 가능합니다. |
| 2.2.5 | ORDER BY 컬럼 가공은 왜 위험한가 | 인덱스 정렬성을 잃습니다. |
| 2.2.6 | SELECT-LIST 가공은 어떤가 | 탐색은 막지 않지만 CPU 비용이 있습니다. |
| 2.2.7 | 자동 형변환은 왜 위험한가 | 보이지 않는 컬럼 가공을 만들 수 있습니다. |
| 2.3.1 | Index Range Scan은 무엇인가 | 조건 범위를 읽는 기본 스캔입니다. |
| 2.3.2 | Index Full Scan은 무엇인가 | 인덱스 전체를 키 순서대로 읽습니다. |
| 2.3.3 | Index Unique Scan은 무엇인가 | Unique 키 단건 탐색입니다. |
| 2.3.4 | Index Skip Scan은 무엇인가 | 선두 컬럼 값을 가정해 후행 컬럼을 탐색합니다. |
| 2.3.5 | Index Fast Full Scan은 무엇인가 | 순서를 포기하고 인덱스 전체를 빠르게 읽습니다. |
| 2.3.6 | Descending Scan은 무엇인가 | 인덱스를 역방향으로 읽습니다. |

### 통합 원리 — 인덱스 튜닝의 두 축 (2.1.1에서 제시)

2장의 모든 절은 인덱스 튜닝의 **두 가지 핵심요소** 중 ①에 해당합니다. ②는 3장의 주제입니다.

```text
인덱스 튜닝
   ├─ ① 인덱스 스캔 효율화  → 2장: 정렬성을 살려 리프 스캔량을 줄임
   │     (컬럼 가공·선두 컬럼 누락·형변환을 피하고, 컬럼 순서를 맞춤)
   └─ ② 랜덤 액세스 최소화  → 3장: 테이블 액세스 횟수를 줄임 ★ 더 중요
```

```text
인덱스 정렬성 활용 (①)
   ↓
시작점과 끝점 결정
   ↓
리프 스캔 범위 축소
   ↓
테이블 랜덤 액세스 축소 (②로 연결)
   ↓
논리적 I/O 감소
```

### 이후 장과의 연결

| 이후 장 | 연결 |
|---|---|
| 3장 | 인덱스 스캔 효율성과 테이블 랜덤 액세스 최소화 |
| 4장 | 조인 시 Inner 테이블 인덱스 탐색 비용 |
| 5장 | 인덱스 정렬성을 이용한 ORDER BY/GROUP BY/Top N 소트 생략 |
| 6장 | DML 시 인덱스 유지 비용 |

## ✅ 2장의 핵심 통찰

1. 인덱스 튜닝은 ① 인덱스 스캔 효율화(2장)와 ② 랜덤 액세스 최소화(3장) 두 축이며, ②가 더 중요합니다.
2. 인덱스는 정렬된 B*Tree이며, 수직 탐색은 시작점 찾기, 수평 탐색은 범위 읽기입니다.
3. 결합 인덱스는 선두 컬럼부터 정렬되므로 선두 컬럼 조건이 핵심입니다.
4. 컬럼 가공과 자동 형변환(LIKE 포함)은 인덱스 정렬성을 깨뜨립니다.
5. 인덱스 스캔 방식마다 정렬성, 범위, I/O 특성이 다릅니다.

📌 한 줄 요약: 2장은 인덱스를 "정렬된 자료구조를 이용해 블록 접근 범위를 줄이는 장치"로 이해하게 만드는 장입니다.
