## 2.2.5 ORDER BY 절에서 컬럼 가공

### 핵심 개념

`ORDER BY` 절에서 인덱스 컬럼을 가공하면 인덱스의 정렬 순서를 사용할 수 없어 소트 연산이 발생할 수 있습니다.

```text
인덱스: col
ORDER BY col
   ↓
인덱스 순서 사용 가능

ORDER BY function(col)
   ↓
원본 col 정렬 순서와 다름
   ↓
별도 Sort 필요
```

### 예시

```sql
order by substr(주문번호, 1, 4)
```

인덱스는 `주문번호` 원본 값 기준으로 정렬되어 있습니다. 하지만 정렬 기준은 `substr` 결과이므로 원본 정렬 순서를 그대로 사용할 수 없습니다.

### 튜닝 방향

정렬 순서를 인덱스로 대체하려면 `ORDER BY`에 인덱스 컬럼을 원형 그대로 사용해야 합니다.

```sql
-- 유리
order by 주문일시

-- 불리 가능
order by to_char(주문일시, 'yyyymmdd')
```

---

## ✅ 이 절(2.2.5)의 핵심 통찰

1. `ORDER BY`에서 컬럼을 가공하면 인덱스 정렬성을 잃을 수 있습니다.
2. 인덱스는 원본 컬럼 값 순서로 정렬되어 있습니다.
3. 소트 생략을 원하면 정렬 컬럼을 원형 그대로 사용해야 합니다.
4. 컬럼 가공은 WHERE뿐 아니라 ORDER BY에서도 위험합니다.

## 🔗 흐름 속 위치

2.2.4의 소트 생략 조건을 깨뜨리는 대표 사례입니다.

📌 한 줄 요약: `ORDER BY`에서 컬럼을 가공하면 정렬된 인덱스를 두고도 다시 정렬해야 할 수 있습니다.
