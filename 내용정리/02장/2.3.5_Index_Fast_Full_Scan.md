## 2.3.5 Index Fast Full Scan

### 핵심 개념

`Index Fast Full Scan`은 인덱스를 테이블처럼 전체 스캔하는 방식입니다. 인덱스 키 순서를 보장하지 않지만, Multiblock I/O와 병렬 처리가 가능해 빠르게 읽을 수 있습니다.

```text
Index Fast Full Scan
   ↓
인덱스 전체 블록을 빠르게 읽음
   ↓
키 순서 보장 없음
```

### Index Full Scan과 차이

| 구분 | Index Full Scan | Index Fast Full Scan |
|---|---|---|
| 읽는 방식 | 키 순서대로 리프 스캔 | 인덱스 세그먼트 전체 스캔 |
| 정렬 순서 | 보장 | 보장 안 함 |
| I/O | 주로 Single Block | Multiblock 가능 |
| 병렬 처리 | 제한적 | 가능 |
| ORDER BY 생략 | 가능 | 불가능 |

### 언제 유용한가

인덱스만 읽고 결과를 만들 수 있고, 정렬 순서가 필요 없을 때 유리합니다.

```text
SELECT 컬럼이 모두 인덱스에 포함
   ↓
테이블 액세스 불필요
   ↓
인덱스가 테이블보다 작음
   ↓
Fast Full Scan 유리
```

---

## ✅ 이 절(2.3.5)의 핵심 통찰

1. Index Fast Full Scan은 인덱스 전체를 빠르게 읽는 방식입니다.
2. 키 순서를 보장하지 않습니다.
3. Multiblock I/O와 병렬 처리가 가능합니다.
4. 인덱스만으로 처리 가능하고 정렬이 필요 없을 때 유리합니다.

## 🔗 흐름 속 위치

Index Full Scan과 혼동하기 쉬운 확장 기능입니다. 정렬성 보장 여부가 핵심 차이입니다.

📌 한 줄 요약: Index Fast Full Scan은 정렬 순서를 포기하고 인덱스 전체를 빠르게 읽는 방식입니다.
