## 2.1.2 인덱스 구조

### 핵심 개념

Oracle B*Tree 인덱스는 루트 블록, 브랜치 블록, 리프 블록으로 구성됩니다. 리프 블록에는 인덱스 키와 테이블 로우를 찾아가기 위한 ROWID가 저장됩니다.

```text
Root Block
   ↓
Branch Block
   ↓
Leaf Block
   ↓
Key + ROWID
```

### B*Tree 구조

| 구성요소 | 역할 |
|---|---|
| 루트 블록 | 탐색의 시작점 |
| 브랜치 블록 | 하위 블록으로 내려가는 경로 안내 |
| 리프 블록 | 실제 인덱스 키와 ROWID 저장 |

리프 블록은 키 순서대로 정렬되어 있고, 서로 연결되어 있습니다. 이 연결 구조 덕분에 시작 지점만 찾으면 옆으로 이동하며 범위 스캔할 수 있습니다.

```text
Leaf A <-> Leaf B <-> Leaf C <-> Leaf D
```

### 인덱스 엔트리

```text
인덱스 키 값 + ROWID
```

ROWID는 테이블 로우가 저장된 위치를 가리킵니다. 따라서 인덱스에서 조건에 맞는 키를 찾은 뒤 ROWID로 테이블 블록을 액세스합니다.

---

## ✅ 이 절(2.1.2)의 핵심 통찰

1. B*Tree 인덱스는 루트, 브랜치, 리프 블록으로 구성됩니다.
2. 리프 블록은 키 순서로 정렬되어 있습니다.
3. 리프 블록에는 키와 ROWID가 저장됩니다.
4. ROWID를 통해 테이블 블록을 랜덤 액세스합니다.

## 🔗 흐름 속 위치

인덱스 튜닝을 이해하려면 먼저 B*Tree 구조를 알아야 합니다. 수직 탐색과 수평 탐색은 이 구조 위에서 작동합니다.

📌 한 줄 요약: 인덱스는 키 순서로 정렬된 B*Tree이며, 리프 엔트리의 ROWID로 테이블을 찾아갑니다.
