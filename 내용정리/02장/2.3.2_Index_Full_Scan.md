## 2.3.2 Index Full Scan

### 핵심 개념

`Index Full Scan`은 인덱스 리프 블록 전체를 키 순서대로 읽는 방식입니다. 테이블 전체가 아니라 인덱스 전체를 읽습니다.

```text
인덱스 왼쪽 끝
   ↓
리프 블록 전체 순차 스캔
   ↓
키 순서 보장
```

### 언제 사용되는가

| 상황 | 설명 |
|---|---|
| 조건절 선두 컬럼이 없지만 인덱스 전체가 유용할 때 | 테이블보다 인덱스가 작을 수 있음 |
| ORDER BY 생략 | 인덱스 순서대로 결과 반환 가능 |
| 인덱스만으로 처리 가능 | 테이블 액세스 없이 결과 생성 가능 |

### Range Scan과 차이

| 구분 | Index Range Scan | Index Full Scan |
|---|---|---|
| 시작점 | 조건에 맞는 지점 | 인덱스 시작 또는 끝 |
| 읽는 범위 | 조건 범위 | 인덱스 전체 |
| 정렬성 | 조건 범위 내 정렬 | 전체 인덱스 키 순서 |

### 주의점

Index Full Scan은 인덱스를 전체 읽는 방식이므로 데이터량이 많으면 비용이 큽니다. 다만 테이블보다 인덱스가 훨씬 작거나 정렬 생략 효과가 크면 선택될 수 있습니다.

---

## ✅ 이 절(2.3.2)의 핵심 통찰

1. Index Full Scan은 인덱스 전체를 키 순서대로 읽습니다.
2. 테이블 Full Scan과 다르며, 인덱스 정렬 순서를 보장합니다.
3. ORDER BY 생략이나 인덱스만 읽고 처리할 때 유용할 수 있습니다.
4. 전체 인덱스를 읽으므로 무조건 빠른 것은 아닙니다.

## 🔗 흐름 속 위치

Range Scan이 특정 범위를 읽는 방식이라면, Full Scan은 인덱스 전체를 정렬 순서대로 읽는 방식입니다.

📌 한 줄 요약: Index Full Scan은 테이블이 아니라 인덱스 전체를 키 순서대로 읽어 정렬성과 작은 크기를 활용하는 방식입니다.
