## 2.3.4 Index Skip Scan

### 핵심 개념

오라클 9i부터 제공하는 스캔 방식으로, **인덱스 선두 컬럼이 조건절에 없어도 인덱스를 활용**합니다. 정확한 적용 조건은:

> **조건절에 빠진 인덱스 선두 컬럼의 Distinct Value 개수가 적고, 후행 컬럼의 Distinct Value 개수가 많을 때 유용하다.**

예: 고객 테이블에서 NDV가 가장 적은 컬럼은 '성별', 가장 많은 컬럼은 '고객번호'.

```sql
-- 인덱스: (성별, 연봉) — 선두 성별이 조건에 없음
select /*+ index_ss(사원 사원_IDX) */ *
from   사원
where  연봉 between 2000 and 4000;
```
```text
Execution Plan
1   TABLE ACCESS (BY INDEX ROWID) OF '사원'
2    INDEX (SKIP SCAN) OF '사원_IDX'
```
힌트: **`index_ss`**(유도) / **`no_index_ss`**(방지)

---

### 작동 원리 — "가능성 있는 리프 블록만 골라 읽기"

> Index Skip Scan은 **루트 또는 브랜치 블록에서 읽은 컬럼 값 정보를 이용해, 조건절에 부합하는 레코드를 포함할 '가능성이 있는' 리프 블록만 골라서 액세스**하는 방식입니다.

`(성별, 연봉)` 인덱스에서 `연봉 between 2000 and 4000`을 찾는다면(그림 2-20):

```text
리프1: 남&800 이하      → 액세스 (남보다 작은 성별이 있을 가능성, 옵티마이저는 모름)
리프2: 남&800 ~ 남&1500  → 2000~4000 없음        → Skip ⏭
리프3: 남&1500 ~ 남&5000 → 2000~4000 포함 가능    → 액세스 ✅
...
여&... 구간도 동일하게 가능성 있는 블록만 골라 읽음
```

선두 컬럼(성별)의 각 값 구간을 **건너뛰며(skip)**, 후행 조건(연봉)을 포함할 가능성이 있는 리프 블록만 방문합니다 — 그래서 'Skip Scan'입니다.

---

### ⚠️ 선두 컬럼이 빠진 경우만이 아니다

**Skip Scan은 선두 컬럼이 없을 때만 작동하는 게 아닙니다.** 더 일반적으로 쓰입니다.

**① 중간 컬럼이 빠진 경우** — 선두는 있고 중간 컬럼만 조건에서 빠질 때도 작동합니다.
```sql
-- 인덱스: 일별업종별거래_PK = (업종유형코드 + 업종코드 + 기준일자)
SELECT /*+ INDEX_SS(A 일별업종별거래_PK) */ 기준일자, 업종코드, 체결건수, ...
FROM   일별업종별거래 A
WHERE  업종유형코드 = '01'                         -- 선두 O
AND    기준일자 BETWEEN '20080501' AND '20080531';  -- 중간(업종코드) 빠짐
```
```text
INDEX (SKIP SCAN) OF '일별업종별거래_PK'
```
- **Range Scan이라면**: `업종유형코드='01'` 구간을 **전부** 스캔해야 함.
- **Skip Scan이라면**: `업종유형코드='01'` 구간에서 `기준일자` 조건을 **포함할 가능성이 있는 리프 블록만** 골라 액세스 → 빠진 중간 컬럼(업종코드)을 건너뜀.

**② NDV 적은 선두 컬럼들이 모두 빠진 경우**도 유용합니다(업종유형코드+업종코드 둘 다 빠지고 기준일자만 조건일 때).

---

### 적합 조건과 한계

| 조건 | 설명 |
|------|------|
| 빠진 선두(또는 중간) 컬럼 NDV가 **적음** | 건너뛸 구간 수가 적어야 함 |
| 후행 조건 컬럼 NDV가 **많음**(선택도 좋음) | 각 구간에서 적은 범위만 읽음 |
| 대안 인덱스 없음 | Range Scan용 인덱스가 없을 때의 차선 |

**한계**: 빠진 컬럼의 NDV가 크면 건너뛸 구간이 너무 많아져, Index Full Scan과 비슷하거나 더 나빠집니다. 수행빈도가 높다면 조건 컬럼이 선두인 인덱스를 따로 만드는 게 낫습니다.

---

## ✅ 이 절(2.3.4)의 핵심 통찰

1. Index Skip Scan은 **빠진 선두 컬럼의 NDV가 적고 후행 컬럼 NDV가 많을 때** 유용하다(`index_ss`/`no_index_ss`).
2. 원리는 **"조건을 포함할 가능성이 있는 리프 블록만 골라 읽고 나머지 구간은 건너뛰는(skip)"** 것이다.
3. **선두 컬럼이 빠질 때만이 아니라, 중간 컬럼이 빠지거나 NDV 적은 여러 선행 컬럼이 빠질 때도** 작동한다.
4. Range Scan은 구간 전체를 읽지만, Skip Scan은 가능성 있는 블록만 읽어 더 효율적일 수 있다.
5. 빠진 컬럼 NDV가 크면 비효율 — 근본 해법은 조건 컬럼을 선두로 한 인덱스 생성이다.

## 🔗 흐름 속 위치

2.2.3에서 "선두 컬럼이 조건절에 있어야 한다"고 했는데, Skip Scan은 그 원칙을 **선두/중간 컬럼이 빠져도 NDV가 적으면 우회**하는 확장 기능입니다. 3.3.7에서 활용 사례를 더 다룹니다.

📌 한 줄 요약: Index Skip Scan은 NDV가 적은 (선두·중간) 컬럼 구간을 건너뛰며 조건을 포함할 가능성이 있는 리프 블록만 골라 읽는, 선두 컬럼 누락 원칙의 우회 기능입니다.
