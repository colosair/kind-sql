## 2.2.6 SELECT-LIST에서 컬럼 가공

### 핵심 개념

SELECT-LIST에서 컬럼을 가공하는 것은 WHERE나 ORDER BY에서 가공하는 것과 성격이 다릅니다. SELECT-LIST 가공은 보통 인덱스 Range Scan 자체를 방해하지 않습니다.

```sql
select to_char(주문일시, 'yyyymmdd')
from 주문
where 주문일시 >= date '2024-06-01'
and 주문일시 <  date '2024-06-02';
```

```text
WHERE 컬럼 원형 유지
   ↓
인덱스 Range Scan 가능
   ↓
조회 후 SELECT-LIST에서 출력 가공
```

### WHERE 가공과 차이

| 위치 | 인덱스 영향 |
|---|---|
| WHERE 컬럼 가공 | Range Scan을 방해할 수 있음 |
| ORDER BY 컬럼 가공 | 소트 생략을 방해할 수 있음 |
| SELECT-LIST 컬럼 가공 | 대체로 탐색 조건에는 영향 없음 |

### 주의점

SELECT-LIST 가공이 항상 공짜는 아닙니다. 많은 로우에 대해 함수를 호출하면 CPU 비용이 증가하고, 인덱스만 읽고 처리할 수 있는 쿼리를 테이블 액세스가 필요한 쿼리로 바꿀 수 있습니다.

```text
가공 대상 로우 많음
   ↓
함수 호출 반복
   ↓
CPU 증가
```

---

## ✅ 이 절(2.2.6)의 핵심 통찰

1. SELECT-LIST 가공은 WHERE 가공보다 인덱스 사용에 덜 치명적입니다.
2. 조건절 컬럼이 원형이면 Range Scan은 가능합니다.
3. 하지만 함수 호출 CPU 비용은 발생합니다.
4. 인덱스만 읽고 처리 가능 여부도 함께 봐야 합니다.

## 🔗 흐름 속 위치

2.2.5와 함께 컬럼 가공 위치에 따른 인덱스 영향 차이를 정리합니다.

📌 한 줄 요약: SELECT-LIST 가공은 탐색을 막지는 않지만, 많은 로우에 반복되면 CPU와 커버링 효과 측면에서 비용이 됩니다.
