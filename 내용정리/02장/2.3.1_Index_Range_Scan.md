## 2.3.1 Index Range Scan

### 핵심 개념

`Index Range Scan`은 인덱스에서 조건에 맞는 시작 지점을 찾은 뒤 리프 블록을 범위 방향으로 스캔하는 가장 기본적인 인덱스 스캔 방식입니다.

```text
수직 탐색
   ↓
시작 리프 엔트리
   ↓
수평 스캔
   ↓
ROWID로 테이블 액세스
```

### 발생 조건

**선두 컬럼이 가공되지 않은 채 조건절에 있어** 정렬된 키의 시작점·종료점을 정할 수 있을 때 발생합니다(2.2.3).

```sql
select * from emp where deptno = 20;
```
```text
Execution Plan
0  SELECT STATEMENT
1   TABLE ACCESS (BY INDEX ROWID) OF 'EMP'
2    INDEX (RANGE SCAN) OF 'EMP_DEPTNO_IDX'    ← B*Tree의 정상적 사용
```

| 조건 | Range Scan |
|------|:---:|
| `where deptno = 20` (등치) | O |
| `where deptno between 10 and 20` (범위) | O |
| `where substr(deptno,1,1)='2'` (선두 가공) | X (Full Scan) |

### 튜닝 포인트

`Index Range Scan`은 이름만으로 효율을 판단할 수 없습니다. 스캔 범위와 테이블 액세스량을 함께 봐야 합니다.

```text
Range Scan
   ↓
리프 범위 좁음 → 효율적
리프 범위 넓음 → 비효율 가능
```

---

## ✅ 이 절(2.3.1)의 핵심 통찰

1. Index Range Scan은 가장 기본적인 인덱스 스캔 방식입니다.
2. 수직 탐색으로 시작점을 찾고 수평 탐색으로 범위를 읽습니다.
3. 선두 컬럼 조건이 중요합니다.
4. 스캔 범위가 넓으면 Range Scan도 느릴 수 있습니다.

## 🔗 흐름 속 위치

2.1~2.2의 구조와 사용 조건을 실제 실행계획 오퍼레이션으로 연결합니다.

📌 한 줄 요약: Index Range Scan은 B*Tree의 시작점으로 내려가 필요한 리프 범위를 옆으로 읽는 방식입니다.
